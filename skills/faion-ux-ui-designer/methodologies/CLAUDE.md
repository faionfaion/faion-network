# UX/UI Methodologies

## Overview

32 core UX methodologies covering usability heuristics, research methods, and design techniques. Each file follows a consistent structure: Problem, Solution, Best Practices, Checklist.

## File Summaries

### Nielsen's 10 Usability Heuristics (M-UX-001 to M-UX-010)

| File | ID | Title | Key Principle |
|------|-----|-------|---------------|
| `M-UX-001-visibility-of-system-status.md` | M-UX-001 | Visibility of System Status | Always keep users informed about what is happening through timely feedback |
| `M-UX-002-match-between-system-and-real-world.md` | M-UX-002 | Match Between System and Real World | Use familiar language, follow real-world conventions, present info logically |
| `M-UX-003-user-control-and-freedom.md` | M-UX-003 | User Control and Freedom | Provide undo/redo, clear exits, support user autonomy |
| `M-UX-004-consistency-and-standards.md` | M-UX-004 | Consistency and Standards | Follow platform conventions, maintain internal consistency |
| `M-UX-005-error-prevention.md` | M-UX-005 | Error Prevention | Eliminate error-prone conditions, provide confirmations for critical actions |
| `M-UX-006-recognition-over-recall.md` | M-UX-006 | Recognition Rather Than Recall | Make objects, actions, options visible; minimize memory load |
| `M-UX-007-flexibility-and-efficiency.md` | M-UX-007 | Flexibility and Efficiency of Use | Accelerators for experts, personalization options |
| `M-UX-008-aesthetic-and-minimalist.md` | M-UX-008 | Aesthetic and Minimalist Design | Remove irrelevant information, prioritize essential content |
| `M-UX-009-error-recovery.md` | M-UX-009 | Help Users Recognize and Recover from Errors | Plain language errors, precise problem indication, constructive solutions |
| `M-UX-010-help-and-documentation.md` | M-UX-010 | Help and Documentation | Searchable help, task-focused, concise steps |

## Usage

Each methodology file includes:
- **Problem**: The challenge the methodology addresses
- **Solution**: How to apply the methodology
- **Best Practices**: Do's and don'ts
- **Checklist**: Verification items

Reference by ID when applying to design work (e.g., "Apply M-UX-005 Error Prevention").
