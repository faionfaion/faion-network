## Key Frameworks

### Pricing Models

| Model | Best For | Characteristics |
|-------|----------|-----------------|
| Flat rate | Simple products | One price, easy to understand |
| Tiered | Feature differentiation | Good/Better/Best structure |
| Usage-based | Variable consumption | Pay-per-use, API calls |
| Per-seat | Team tools | Price per user |
| Freemium | High volume, PLG | Free tier + paid upgrade |
| Hybrid | Complex SaaS | Base + usage components |

### Unit Economics

| Metric | Formula | Target |
|--------|---------|--------|
| CAC | Total Acquisition Cost / New Customers | Lower is better |
| LTV | ARPU x Avg Lifespan | Higher is better |
| LTV:CAC | LTV / CAC | 3:1 or higher |
| Payback | CAC / Monthly Revenue | < 12 months |
| Churn | Lost Customers / Start Customers | < 5% monthly |
| NRR | (Start MRR + Expansion - Churn) / Start MRR | > 100% |

### Churn Prevention Signals

| Signal | Risk Level | Action |
|--------|------------|--------|
| No login 14+ days | High | Personal outreach |
| Support complaints | Medium | CS escalation |
| Feature underuse | Medium | Education email |
| Payment failure | High | Dunning sequence |
| Downgrade request | High | Retention call |

## Legal Compliance Quick Reference

| Regulation | Scope | Key Requirements |
|------------|-------|------------------|
| GDPR | EU users | Consent, data access, deletion rights |
| CCPA | California | Disclosure, opt-out, data deletion |
| PCI-DSS | Payment data | Secure handling, encryption |
| COPPA | Children < 13 | Parental consent |
| CAN-SPAM | Email marketing | Unsubscribe, sender identity |
