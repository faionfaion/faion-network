# Agent Integration — Code Review

## When to use
- All PRs in a multi-engineer team — code review is the cheapest defect-detection layer.
- Before merging LLM-authored PRs — agents pattern-match well but produce subtly wrong code (hallucinated APIs, missed edge cases). A second agent or human review catches it.
- Security-sensitive paths (auth, payment, IAM, crypto) — mandatory pre-merge dual review.
- Libraries / SDKs where API surface is sticky — review enforces backward compatibility.
- Onboarding ramps — junior engineer code reviewed by senior + an automated reviewer to teach style + patterns.

## When NOT to use
- Trunk-based solo prototypes pre-MVP — review overhead outpaces signal. Ship, then PR-review post-MVP.
- Mechanical refactors generated by codemod with green CI + 100% test coverage on touched code — skim, don't deep-review.
- Vendored / generated code (proto stubs, OpenAPI clients) — review the generator config, not the output.
- Documentation typos — accept fast; don't gate.

## Where it fails / limitations
- **Reviewer fatigue.** PRs >400 lines get rubber-stamped. Force budget: <400 changed lines per PR; agents enforce via `pr-size` check.
- **Bikeshedding.** Style debates eat time when the linter could decide. Push style to formatter (Prettier, ruff format, gofmt) — review only logic.
- **LGTM culture.** Reviewers approve to unblock without engaging. Symptom: zero review comments. Counter: required review templates / checklists.
- **No ownership signals.** PR routes to anyone, nobody owns. Use `CODEOWNERS` so reviews auto-assign by path.
- **Agent reviews lack context.** LLM reviewers don't know the team's prior decisions; flag false positives ("use type hints" on a project that doesn't). Configure with project-style guide before reviewing.
- **Review-after-merge antipattern.** GitHub allows; teams adopt → defects ship. Enforce branch protection: `required_pull_request_reviews=1`.
- **Long-living branches.** PRs sit a week, conflict, get force-pushed; reviewers re-review from scratch. Time-bound: stale-after-72h, auto-close-after-14d.
- **Security review skipped.** Reviewer can't spot SSRF / IDOR. Add automated scanners (Semgrep, CodeQL) as separate gate before human review.

## Agentic workflow
Layered review pipeline: (1) **automated gate** runs lint + format + type + test + security scan + coverage diff before human eyes; (2) a **review-summary** subagent posts a 200-token summary of the PR (what changed, scope, risks) to free reviewers from spelunking; (3) an **agent-reviewer** subagent comments on diff hunks for: missed null checks, hallucinated APIs, untested branches, naming inconsistencies; (4) **human reviewer** focuses on architecture / API / business logic — the things agents can't judge. Block merge until: green CI, ≥1 human approval, all conversations resolved. The agent-reviewer must be configured with project conventions (`.agents/review-rubric.md`) to avoid false positives.

### Recommended subagents
- `faion-sdd-executor-agent` (`agents/faion-sdd-executor-agent.md`) — quality gate runs the full pre-merge suite (`lint && test && coverage --fail-under=80 && security-scan`).
- A purpose-built **pr-summary-agent** (worth creating): posts auto-summary on PR open; updates on push.
- A **diff-reviewer-agent** (worth creating): posts inline review comments using project rubric; never approves, only flags.
- A **changelog-checker-agent** (worth creating): blocks PRs missing a `## [Unreleased]` entry per project convention (this repo's pre-commit hook enforces it).
- A **secret-scanner-agent** wired to `gitleaks` / `trufflehog` — pre-merge gate, not advisory.
- `password-scrubber-agent` (`agents/password-scrubber-agent.md`) — final pass before merge to scrub committed `.env` / config snippets.
- `security-review` (skill, available in this repo): runs full security review of pending changes on the current branch.

### Prompt pattern
PR summary:
```
You are a senior reviewer. Read the PR diff and post a comment
with these sections:
- Scope: 1 sentence.
- Files changed: count + risky files (auth/, payments/, migrations/).
- Tests: which behaviors covered? gaps?
- Risks: list each non-obvious risk + line reference.
- Out of scope: anything that should be a follow-up PR.
Do NOT approve.
Do NOT request changes unless the diff is unbuildable.
```

Diff review (rubric-driven):
```
Review the diff against .agents/review-rubric.md.
For each hunk, output:
- file:line
- severity: blocker | major | minor | nit | praise
- finding (1 sentence)
- suggested patch (≤5 lines)
Skip style issues — Prettier/ruff handles those.
Stop after 30 findings; ask for follow-up if more.
```

## Templates & scripts
See `templates.md` for PR description template, GitHub workflow yamls, review checklists. Add a quick reviewer-load balancer (≤50 lines):

```bash
#!/usr/bin/env bash
# pr-balance.sh — assign least-loaded reviewer from CODEOWNERS.
# Usage: pr-balance.sh <PR_NUMBER>
set -euo pipefail
PR="${1:?PR number required}"
REPO="$(gh repo view --json nameWithOwner -q .nameWithOwner)"
OWNERS_RAW="$(gh api repos/$REPO/contents/CODEOWNERS -q .content | base64 -d)"
# Pull unique GH usernames from CODEOWNERS (skip teams)
CANDIDATES=$(echo "$OWNERS_RAW" | grep -oE '@[A-Za-z0-9_-]+' \
             | sed 's/@//' | grep -v '/' | sort -u)
LOWEST=""
LOWEST_COUNT=99999
for user in $CANDIDATES; do
  N=$(gh search prs --repo "$REPO" --review-requested "$user" \
                    --state open --json number -q '. | length')
  echo "$user: $N pending"
  if (( N < LOWEST_COUNT )); then
    LOWEST_COUNT=$N
    LOWEST="$user"
  fi
done
[ -n "$LOWEST" ] || { echo "no candidates"; exit 1; }
gh pr edit "$PR" --add-reviewer "$LOWEST"
echo "assigned $LOWEST ($LOWEST_COUNT pending)"
```

Hook into `gh pr create` post-create automation.

## Best practices
- **<400 lines / <20 files per PR.** Above that, defect detection drops sharply (industry data: SmartBear, Cisco studies).
- **PR template required.** Description / linked issue / test evidence / screenshots — gates accidental empty descriptions.
- **CI must be green before review.** Reviewers don't debug build failures.
- **Conversation = blocker until resolved.** Author resolves; reviewer re-checks. No "ack" merges.
- **Review for: correctness, design, tests, security, naming, observability.** Skip style (formatter), spelling (linter), nits without impact.
- **Limit "request changes" to actual blockers.** Use "comment" for nits. Reduces back-and-forth.
- **Self-review first.** Author opens own diff in GH, walks through it, comments on weird parts. Cuts review time 40%.
- **Time-box the review.** 30-60 min per session; resume after break for large PRs.
- **CODEOWNERS for routing.** Auto-assigns by path; agents bias toward calmer humans only on critical paths.
- **Async by default, sync for stuck threads.** A 5-min huddle resolves what 2 days of comments don't.
- **Praise + suggest patches.** Comments without patches double the loop. "Suggested change" buttons on GitHub speed merges.

## AI-agent gotchas
- **Approving without engagement.** Agent reviewer hits "Approve" on green CI. Configure to comment-only; never auto-approve.
- **Drowning the diff in comments.** Agent flags 100 minor issues; humans tune out. Cap output (top 30 by severity).
- **Hallucinating "missing" code.** LLM "notices" a function isn't defined — but it's imported from another file. Always include neighboring file context.
- **Style-as-bug.** Agent flags `let` vs `const` though linter ignores. Wire agent to read `.eslintrc` / `pyproject.toml` first.
- **Not reading the PR description.** Agent flags "untested" code that the PR description explains as a follow-up. Feed PR description into prompt.
- **Test-coverage false positive.** Agent says "no tests" when tests live in `__tests__/` it didn't see. Always pass full diff scope.
- **Re-reviewing identical force-push.** Agent re-reviews from scratch on every push. Cache prior review by commit SHA; only diff new commits.
- **Cross-PR context blindness.** Agent flags inconsistency that's deliberately rectified in a follow-up PR. Show open PRs in same module.
- **Secret leaks in suggested patches.** Agent suggests committing a value from `.env.example`. Always pass through secret-scanner before posting.
- **Approve from same author.** Agent-as-author + agent-as-reviewer = no real review. Enforce branch protection: `dismiss_stale_reviews_on_push=true`, `require_code_owner_reviews=true`, `required_review_count >= 1` from a different identity.
- **Time-of-check / time-of-use.** Agent reviews HEAD~3 because of stale checkout. Always re-fetch latest before generating comments.
- **Suggesting refactors that break public API.** Agent doesn't know `function foo` is exported in a published package. Keep a `public-api.md` and feed it.
- **Ignoring the `## [Unreleased]` requirement.** Project hooks block commits without a CHANGELOG entry; agents push and discover at hook fail. Enforce in agent prompt explicitly.
