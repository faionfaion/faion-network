# Constitution Template

Шаблон для `constitution.md` проекту.

---

## Структура документа

```
1. Метадані (версія, статус, дата)
2. Огляд проекту
3. Технологічний стек
4. Структура проекту
5. Архітектурні принципи
6. Стандарти коду
7. Тестування
8. Заборонені практики
9. Code Review Checklist
10. Історія змін
```

---

## Обов'язкові розділи

- **Метадані** — версія, статус, дата
- **Огляд** — що це за проект
- **Технологічний стек** — мови, фреймворки, інфра

## Рекомендовані розділи

- **Структура проекту** — директорії, модулі
- **Архітектурні принципи** — патерни, діаграми
- **Стандарти коду** — naming, linters, formatting
- **Тестування** — coverage, типи тестів

## Опціональні розділи

- API Standards
- Database Guidelines
- Security
- Git Workflow
- Logging & Monitoring
- Performance
- Заборонені практики
- Code Review Checklist

## Приклади секцій

### Заборонені практики (приклад)

```markdown
| Практика | Причина |
|----------|---------|
| `any` type | Втрата type safety |
| Бізнес-логіка в UI | Порушує тестованість |
| `.only` в тестах | Блокує CI |
| Хардкод конфігурації | Ускладнює deployment |
| `console.log` в production | Забруднює логи |
```

### Code Review Checklist (приклад)

```markdown
Архітектура:
[ ] Відповідає шарам архітектури
[ ] Немає циклічних залежностей
[ ] Використані правильні патерни

Код:
[ ] Типи явно визначені
[ ] Іменування відповідає конвенціям
[ ] Немає дублювання коду

Тести:
[ ] Покриття >= мінімуму
[ ] Edge cases протестовані
[ ] Немає .only/.skip
```
