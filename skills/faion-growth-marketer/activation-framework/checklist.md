# Checklist - Activation Framework

## Define Activation

- [ ] Analyze user cohorts and retention patterns
- [ ] Identify actions that correlate with D30 retention
- [ ] Choose one key activation metric (not vanity metrics)
- [ ] Verify metric difference between retained vs churned users
- [ ] Set activation benchmark (target: 30-50%)
- [ ] Document activation definition

## Measure Current State

- [ ] Calculate current activation rate
- [ ] Measure time to activation (TTA)
- [ ] Segment users by acquisition channel
- [ ] Analyze activation by user cohort
- [ ] Identify users who don't activate
- [ ] Create baseline metrics dashboard

## Map User Journey

- [ ] List all steps from sign-up to activation
- [ ] Identify friction points
- [ ] Measure conversion at each step
- [ ] Find the biggest dropout point
- [ ] Document current user flow
- [ ] Create journey diagram

## Design Activation Experience

- [ ] Simplify onboarding flow
- [ ] Reduce steps to first activation
- [ ] Create targeted tutorials for user type
- [ ] Personalize onboarding based on use case
- [ ] Design in-app guidance/tooltips
- [ ] Create email activation sequence

## Implementation & Testing

- [ ] Build new onboarding experience
- [ ] A/B test onboarding improvements
- [ ] Test alternative flows (personalized paths)
- [ ] Test timing of activation prompts
- [ ] Test messaging variations
- [ ] Monitor activation rate impact

## Optimization Rounds

- [ ] Analyze experiment results
- [ ] Measure impact on activation rate
- [ ] Track time to activation improvement
- [ ] Verify retention impact (should increase)
- [ ] Run follow-up experiments
- [ ] Continuously improve weak points

## Monitoring & Maintenance

- [ ] Create activation dashboard
- [ ] Set alert for activation rate drops
- [ ] Weekly activation rate review
- [ ] Monitor by channel/cohort variations
- [ ] Track seasonal patterns
- [ ] Iterate on onboarding continuously
