# faion-improver

Session-based continuous improvement workflow.

## When to Use

- After completing a significant task that produced new knowledge
- When the user asks to audit, review, or improve their system
- When patterns or mistakes should be captured for future sessions
- When experience should become a reusable skill

## Workflow

```
Investigate → Classify → Brainstorm → Apply → Log → Commit → Skill
```

## Methodologies (5)

- **system-audit**: Parallel investigation of configs, security, performance, DX
- **gap-analysis**: Classify findings by priority (CRITICAL → LOW)
- **improvement-application**: Safe fix application with backups
- **knowledge-capture**: Logging to improvement-log.md, .aidocs/memory/, auto-memory
- **skill-extraction**: Creating new faion- skills from domain experience

## Related Skills

- `faion-brainstorm` (multi-agent brainstorming)
- `faion-server-craft` (server config/tuning)
- `faion-devops-engineer` (infrastructure)
