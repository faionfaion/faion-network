# Topical Authority Checklist

## Phase 1: Topic Analysis & Planning

### Topic Selection
- [ ] Identify 5-10 core topics for your business
- [ ] Research topic breadth (how many subtopics?)
- [ ] Analyze competitor coverage depth
- [ ] Assess market demand for each topic
- [ ] Prioritize by business relevance
- [ ] Document topic objectives
- [ ] Set authority-building timeline

### Subtopic Mapping
- [ ] For each topic, list all major subtopics
- [ ] Research questions people ask about topic
- [ ] Document edge cases/niche areas
- [ ] Identify related but different topics
- [ ] Map subtopic relationships
- [ ] Create hierarchy (pillar â†’ clusters)
- [ ] Identify content gaps

### Competitor Analysis
- [ ] Identify top 5 competitors ranking for pillar
- [ ] Analyze their content on topic
- [ ] List all subtopics they cover
- [ ] Identify missing subtopics
- [ ] Assess content depth
- [ ] Note internal linking strategies
- [ ] Document gaps you can fill

## Phase 2: Pillar Page Creation

### Pillar Page Planning
- [ ] Define pillar topic (broad, main keyword)
- [ ] Research pillar search volume and difficulty
- [ ] Plan target word count (3,000+ words)
- [ ] Outline major sections
- [ ] List subtopics to mention
- [ ] Plan internal linking to clusters
- [ ] Identify external sources to cite

### Pillar Content Development
- [ ] Write comprehensive pillar page
- [ ] Cover entire topic breadth
- [ ] Answer primary keyword intent
- [ ] Include overview of all subtopics
- [ ] Link to all cluster pages
- [ ] Use clear heading hierarchy
- [ ] Optimize for user navigation

### Pillar Page Optimization
- [ ] Optimize title tag (primary keyword)
- [ ] Write compelling meta description
- [ ] Use H1 with primary keyword
- [ ] Create H2s for major sections
- [ ] Add internal links to cluster pages
- [ ] Include related external links
- [ ] Implement schema markup (Article)

## Phase 3: Cluster Page Creation

### Cluster Content Planning
- [ ] For each major subtopic, create cluster page
- [ ] Research subtopic keywords
- [ ] Plan word count (1,500-2,500 words)
- [ ] Outline main sections
- [ ] List related subtopics
- [ ] Identify primary sources
- [ ] Plan internal linking strategy

### Cluster Content Development
- [ ] Write detailed cluster pages
- [ ] Cover subtopic comprehensively
- [ ] Answer related questions
- [ ] Target long-tail keywords
- [ ] Provide specific, actionable value
- [ ] Include examples/case studies
- [ ] Reference pillar page
- [ ] Link to related clusters

### Cluster Page Optimization
- [ ] Optimize title tag (target keyword)
- [ ] Write meta description
- [ ] Use H1 with target keyword
- [ ] Create H2s for sections
- [ ] Link back to pillar page
- [ ] Cross-link to related clusters
- [ ] Implement schema markup

## Phase 4: Internal Linking Structure

### Linking Architecture
- [ ] Create visual diagram of structure
- [ ] Pillar links to all clusters
- [ ] Clusters link back to pillar
- [ ] Clusters link to related clusters
- [ ] Cross-topical links (where relevant)
- [ ] Homepage links to pillars
- [ ] Category pages link appropriately

### Anchor Text Strategy
- [ ] Use keyword-rich anchor text (natural)
- [ ] Vary anchor text (not exact match only)
- [ ] Include brand terms
- [ ] Include question-based anchors
- [ ] Avoid over-optimization
- [ ] Keep links contextual
- [ ] Test for natural reading

### Link Distribution
- [ ] Equal link distribution to similar clusters
- [ ] No orphan pages
- [ ] Maximum 3 clicks from homepage
- [ ] Important pages have more internal links
- [ ] Natural link patterns
- [ ] Regular internal link maintenance

### Topic Coverage

## Phase 6: Authority Building

### Backlink Strategy
- [ ] Develop linkable assets (guides, tools, research)
- [ ] Promote pillar page aggressively
- [ ] Build links to pillar first
- [ ] Then build links to clusters
- [ ] Focus on quality over quantity
- [ ] Get links from topic-related sites
- [ ] Diversify link sources

### Brand Mentions & Citations

- [ ] Get quoted as expert on topic
- [ ] Participate in industry discussions
- [ ] Contribute guest posts on topic
- [ ] Appear in roundup articles
- [ ] Get mentioned in research
- [ ] Build expert authority

### Entity Optimization
- [ ] Claim/optimize entity knowledge panels
- [ ] Ensure consistent entity naming
- [ ] Link between entity pages
- [ ] Get entity mentioned by authoritative sites
- [ ] Build topic entity relationships
- [ ] Optimize for knowledge graphs
