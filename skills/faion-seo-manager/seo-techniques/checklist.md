# SEO Techniques Checklist

## Phase 1: AI Search Optimization (AEO)

### llms.txt Setup
- [ ] Create /llms.txt file at domain root
- [ ] Define core pages (about, pricing, docs)
- [ ] List product/service pages
- [ ] Set citation format preferences
- [ ] Mark important pages for crawling
- [ ] Exclude premium/paywalled content
- [ ] Exclude admin and internal sections
- [ ] Test file accessibility

### AI-Friendly Content Structure
- [ ] Clear entity definitions in first paragraph
- [ ] Consistent entity naming throughout
- [ ] Entity relationships explicit (e.g., "Author X works at Company Y")
- [ ] Statistics attributed to sources
- [ ] Dates on all time-sensitive content
- [ ] Author credentials and expertise visible

### Structured Data for LLMs
- [ ] Organization schema with credentials
- [ ] Article schema with author/date
- [ ] Person schema for experts/authors
- [ ] BreadcrumbList for topic relationships
- [ ] FAQSchema for Q&A content
- [ ] CreativeWork schema for case studies

## Phase 2: Schema Markup Implementation

### Core Schema Setup
- [ ] Organization schema (global, header/footer)
- [ ] Article/NewsArticle schema (blog posts)
- [ ] Author schema linked to articles
- [ ] BreadcrumbList schema (navigation)
- [ ] LocalBusiness schema (if applicable)
- [ ] Product schema (e-commerce)
- [ ] Person schema (team/expert pages)

### Advanced Schema
- [ ] FAQSchema with Q&A pairs
- [ ] ScholarlyArticle for research papers
- [ ] BlogPosting for blog content
- [ ] NewsArticle for news items
- [ ] WebPage schema for main pages
- [ ] Collection schema for grouped content

### Schema Testing & Validation
- [ ] Valid JSON-LD syntax (no errors)
- [ ] Google Rich Results Test passing
- [ ] schema.org validator passing
- [ ] Multiple schema types working together
- [ ] No conflicting schema definitions
- [ ] Mobile-friendly schema rendering

## Phase 3: Internal Linking Strategy

### Pillar & Cluster Structure
- [ ] Identify 5-10 pillar topics
- [ ] Create pillar pages (comprehensive guides)
- [ ] Create cluster pages (specific subtopics)
- [ ] Map cluster pages to pillars
- [ ] Document linking structure

### Internal Linking Implementation
- [ ] Pillar pages link to all cluster pages
- [ ] Cluster pages link back to pillar
- [ ] Related content cross-links
- [ ] Keyword-rich anchor text (natural)
- [ ] No orphan pages (all have internal links)
- [ ] Maximum 3 clicks from homepage

### Link Equity Distribution
- [ ] Homepage links to all pillar pages
- [ ] Blog hub links to category posts
- [ ] Old posts updated with new links
- [ ] New posts link to relevant old content
- [ ] Footer links to important pages
- [ ] Natural link distribution

## Phase 4: Social Meta Tags

### Open Graph Implementation
- [ ] og:title (same as page title)
- [ ] og:description (compelling summary)
- [ ] og:image (1200x630px recommended)
- [ ] og:image:alt (describe image)
- [ ] og:url (canonical URL)
- [ ] og:type (website/article/etc.)
- [ ] og:locale (language)

### Twitter Card Setup
- [ ] twitter:card type specified (summary/large_image)
- [ ] twitter:title (max 70 chars)
- [ ] twitter:description (max 200 chars)
- [ ] twitter:image (min 506x506px)
- [ ] twitter:creator (author handle)
- [ ] twitter:site (brand account)
- [ ] Twitter Card validated

### LinkedIn Meta Tags
- [ ] Company page link
- [ ] Article author specified
- [ ] Publication date explicit
- [ ] Keywords relevant
- [ ] Description optimized
- [ ] Image for sharing

## Phase 5: Content Optimization

### Semantic HTML
- [ ] Single H1 per page
- [ ] Proper H2-H6 hierarchy
- [ ] Semantic elements (header, main, footer, article)
- [ ] Proper list markup (ul, ol, li)
- [ ] Table markup for tabular data
- [ ] Code snippets with proper tags
- [ ] Emphasis on important terms

### Entity & Relationship Markup
- [ ] Key entities bold on first mention
- [ ] Related entities linked
- [ ] Author credentials visible
- [ ] Publication/modification dates
- [ ] Source attribution for quotes/data
- [ ] Related topics cross-referenced

### Content Freshness
- [ ] Identify outdated content
- [ ] Update statistics and data
- [ ] Review and refresh yearly
- [ ] Update "last modified" dates
- [ ] Add new information
- [ ] Remove deprecated content
- [ ] Archive old versions if needed

## Phase 6: Knowledge Graph Integration

### Entity Optimization
- [ ] Consistent entity naming across site
- [ ] Entity descriptions comprehensive
- [ ] Entity relationships explicit
- [ ] Backlinks to entity pages
- [ ] Brand mentions tracked
- [ ] Citation relationships defined

### Knowledge Base Structure
- [ ] Create entity pages (if applicable)
- [ ] Define relationships between entities
- [ ] Create disambiguation pages
- [ ] Link to authoritative sources
- [ ] Include external citations
- [ ] Monitor Knowledge Graph appearance

## Phase 7: Accessibility for Content

### Content Accessibility
- [ ] Proper heading structure for navigation
- [ ] Alt text on all images (descriptive)
- [ ] Links have descriptive text (not "click here")
- [ ] Color not sole means of conveying info
- [ ] Content readable without CSS
- [ ] Videos have captions and transcripts
- [ ] Downloadable files accessible

### Semantic Clarity
- [ ] Language clearly specified (lang="en")
- [ ] Abbreviations defined on first use
- [ ] Complex terms explained
- [ ] Jargon minimized or explained
- [ ] Lists properly structured
- [ ] Tables have proper headers

- [ ] Validate all JSON-LD syntax

- [ ] Test rich snippets with Rich Results Test
- [ ] Check hreflang implementation
- [ ] Verify schema markup correctness

### Post-Launch Monitoring
- [ ] Monitor Search Console for errors
- [ ] Check rich snippet appearance
- [ ] Monitor Knowledge Graph
- [ ] Track CTR from SERPs
- [ ] Review snippet appearance
- [ ] Monitor for schema issues

### Tracking Setup
- [ ] Track rich snippet impressions
- [ ] Monitor CTR from rich snippets
- [ ] Track referral sources
- [ ] Monitor social shares (optional)
- [ ] Track internal link clicks
- [ ] Document content performance

### Optimization
- [ ] Identify underperforming content
- [ ] Update low-CTR snippets
- [ ] Improve metadata for better visibility
- [ ] Test different open graph images
- [ ] Refine social descriptions
- [ ] A/B test title variations
