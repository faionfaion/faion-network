# Productized service launch (4-week sprint)

## Intent

Convert one repeated client problem into a fixed-price, fixed-scope offer with a landing page, a booking flow, and 3 paid pilots — moving the freelancer from per-hour to per-outcome billing.

## Scope

Convert one repeated client problem into a fixed-price, fixed-scope offer with a landing page, a booking flow, and 3 paid pilots — moving the freelancer from per-hour to per-outcome billing.

## Stages

### 1. Find the repeated pattern

Pick the one problem you have solved 3+ times.

Tasks:
- Audit last 12 months of client work
- Identify problems you have repeated
- Pick the most lucrative + repeatable one

Outputs:
- Client work audit
- Repeated problem list
- Chosen problem

Decision gate: Advance only when problem has been solved 3+ times.

### 2. Productize the offer

Fixed scope, fixed price, fixed timeline.

Tasks:
- Define the deliverable list (no flex)
- Set a single fixed price
- Lock a fixed delivery window (e.g. 2 weeks)

Outputs:
- Productized canvas
- Price card
- Delivery window

Decision gate: Advance only when offer fits on one page.

### 3. Delivery system

Build the SOPs that make it repeatable.

Tasks:
- Document every delivery step
- Create templates for common artifacts
- Wire automation for repetitive parts

Outputs:
- Delivery SOP
- Template library
- Automation tickets

Decision gate: Advance only when SOP can be executed without your custom thinking.

### 4. Landing page

A landing built around outcomes, not hours.

Tasks:
- Write the landing copy from the customers desired outcome
- Add proof: testimonials + case studies
- Wire booking flow with calendar

Outputs:
- Landing live
- Proof section
- Booking flow

Decision gate: Advance only when booking flow completes end-to-end.

### 5. Pre-sell pilots

Land 3 paid pilots before the public push.

Tasks:
- Outreach to warm past clients first
- Offer pilot pricing with a defined scope
- Close 3 paid pilots

Outputs:
- Pilot outreach log
- Pilot pricing terms
- 3 signed pilots

Decision gate: Advance only with 3 paid pilots in the pipeline.

### 6. Run the pilots

Deliver flawlessly with the SOP.

Tasks:
- Execute the SOP for each pilot
- Capture friction + improvements
- Get written testimonials at the end

Outputs:
- 3 pilot deliveries
- Friction log
- Testimonials

Decision gate: Advance only when all 3 pilots complete with positive testimonial.

### 7. Refine + public launch

Tighten the offer, then go public.

Tasks:
- Tighten SOP based on pilot feedback
- Update landing with new proof
- Push outbound + content to ICP

Outputs:
- Refined SOP
- Updated landing
- Outbound + content push log

Decision gate: Advance only when public launch shipped to ICP.

### 8. Switch billing model

Stop selling hours, start selling outcomes.

Tasks:
- Wind down hourly client contracts gracefully
- Re-route new inquiries to the productized offer
- Update bio + LinkedIn + Upwork

Outputs:
- Hourly wind-down plan
- Routing rules
- Profile updates

Decision gate: Cycle closes when new inquiries default to the productized offer.

### 9. Lessons + iterate

Update the playbook based on real revenue.

Tasks:
- Tally revenue per delivery
- Compare to hourly equivalent
- Decide: scale, raise price, or sunset

Outputs:
- Revenue tally
- Hourly comparison
- Next-move decision

Decision gate: Cycle closes when next-move is committed in writing.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
