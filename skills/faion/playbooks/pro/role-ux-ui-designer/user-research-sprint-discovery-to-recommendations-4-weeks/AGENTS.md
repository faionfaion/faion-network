# User research sprint: discovery to recommendations (4 weeks)

## Context

UX research lead runs a 4-week closed-loop discovery on one product area. Includes scoping + plan sign-off, mixed-methods fieldwork (interviews + survey + contextual inquiry), synthesis with affinity diagramming, and a stakeholder readout that hands a prioritized opportunity tree to design and PM. Done when stakeholders accept the opportunity tree and at least one opportunity is queued in the product backlog.

## Outcome

Open product question -> prioritized opportunity tree ready for design/PM to action. Closed-loop 4-week discovery cycle for one product area: signed plan, fieldwork, synthesis, prioritized opportunity tree.

## Steps

1. Lock the research question and method mix before recruiting. Write a 1-page research brief: question, decisions it informs, audience; Pick method mix (interviews + survey + contextual); Get sponsor sign-off before recruiting
2. The right participants, in time, with consent. Define screener criteria from audience segments; Recruit 8-12 interview participants + survey panel; Confirm consent + compensation
3. Mixed-methods data collection across the cohort. Run interviews + contextual inquiry sessions; Launch survey + open responses; Optional: short diary study where behaviour over time matters
4. Code the qualitative data fast without losing signal. Run AI transcript analysis for pattern detection; Human-review the AI codes; reject false positives; Cross-reference survey results to confirm patterns
5. Convert insights into a structured opportunity tree. Map insights to outcomes, opportunities, solutions; Rank opportunities by impact + confidence + effort; Define success metrics per top opportunity
6. Stakeholders accept and use the findings. Run a 1h readout with sponsor + design + PM; File full report + raw artifacts in the repo; Queue at least one opportunity in the product backlog

## Decision points

- **Scope + plan** -> Advance only after sponsor sign-off.
- **Recruit** -> Advance when target sample is confirmed.
- **Fieldwork** -> Advance only when sample is sufficient (saturation on top themes).
- **AI-assisted analysis** -> Advance only when each insight has >=3 evidence points.
- **Opportunity tree** -> Advance after design + PM can articulate the top 3 opportunities.
- **Readout + handoff** -> Done when sponsor accepts and a backlog entry exists.
