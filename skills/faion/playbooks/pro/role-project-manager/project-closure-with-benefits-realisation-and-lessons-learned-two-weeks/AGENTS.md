# Project closure with benefits realisation + lessons learned (two weeks)

## Context

Hand over a finished project cleanly: scope verified, contracts closed, handover docs delivered, retrospective captured, benefits-realisation report sent, team released, account warm for next engagement.

Tier: **pro**. Complexity: **medium**. Group: **role-project-manager**. Persona: **role-project-manager**.

## Outcome

This playbook is done when:

- Every scope item has written accept/reject.
- Final quality + EVM reports signed.
- Benefits-realisation report confirmed by client.
- Handover pack signed by receiving team.
- 60-day follow-up call scheduled.

## Steps

### 1. Scope verification

Confirm every deliverable was delivered.

Tasks:
- Walk the scope register with the client.
- Sign off on each accepted deliverable.
- Document any scope variances + their rationale.

Outputs:
- Signed scope acceptance log.

Decision gate: Advance when every scope item has a written accept/reject.

### 2. Quality + cost reconciliation

Reconcile what we delivered vs what we baselined.

Tasks:
- Run a final quality review.
- Reconcile EVM final vs baselined cost.
- Note the variance + cause.

Outputs:
- Final quality report.
- EVM closeout report.

Decision gate: Advance when both reports are signed by PM + delivery lead.

### 3. Benefits realisation

Tell the client what they actually got.

Tasks:
- Pull the originally promised benefits.
- Measure what was actually realised.
- Write a benefits-realisation report.

Outputs:
- Benefits-realisation report.

Decision gate: Advance when the client confirms the benefits picture.

### 4. Contracts + procurement close

Close every commercial obligation.

Tasks:
- Close out subcontractor and vendor commitments.
- Confirm final invoice + payment terms.
- Archive contracts per retention policy.

Outputs:
- Closed contracts log.

Decision gate: Advance when every commercial obligation has a close status.

### 5. Handover docs

Hand the running system to its new owner.

Tasks:
- Build the handover pack (architecture, ops, decisions, gotchas).
- Walk the pack with the receiving team.
- Get receiving-team sign-off.

Outputs:
- Signed handover pack.

Decision gate: Advance when receiving team signs the handover.

### 6. Lessons learned

Capture the lessons before the team disperses.

Tasks:
- Run a structured retro with the delivery team.
- Tag lessons as 'do more', 'do less', 'do differently'.
- File lessons into the org knowledge base.

Outputs:
- Lessons-learned doc.

Decision gate: Advance when at least 5 actionable lessons are captured.

### 7. Team release + account warmth

Release the team and keep the door open.

Tasks:
- Release team back to bench / next engagement.
- Send a closing note to the client sponsor.
- Schedule a 60-day follow-up call.

Outputs:
- Team release log.
- Closing client note.

Decision gate: Required: a 60-day follow-up call on the calendar.

## Decision points

- Realised vs intended benefits — report both honestly; do not pad realised numbers.
- Hard close vs soft close — soft close (60-day follow-up) for warm accounts; hard close for one-off engagements.
