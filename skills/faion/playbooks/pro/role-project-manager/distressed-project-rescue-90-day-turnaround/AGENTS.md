# Distressed-project rescue (90-day turnaround)

## Context

A project has been flagged red by the client or steerco. This playbook is the 90-day sequence the PM runs: diagnose root causes, stabilize the burn, communicate transparently, deliver visible wins, and re-baseline. Done when status is green for 3 consecutive weekly reports and the sponsor signs the remediation acceptance.

## Outcome

Red status -> three consecutive green weekly reports + signed remediation acceptance from sponsor. Defined sequence to diagnose, stabilize, and restore confidence on a red-flagged project before contract loss.

## Steps

1. Find the real root causes, not the symptoms. Interview core team + key client stakeholders (5-7 conversations); Map current risk register against actuals; Rank root causes by impact + reversibility
2. Stop the bleeding before re-planning. Define a 2-week stabilization plan with concrete cut-overs; Get sponsor agreement on scope freezes; Stand up rapid-response triage for crit-path issues
3. Build a credible 90-day path back to green. Re-baseline scope, schedule, budget against the new reality; Apply AI-assisted risk aging to surface near-term threats; Lock weekly traffic-light criteria
4. Ship visible wins each week. Daily 15-min crit-path standup; Weekly demos to sponsor (even if partial); Track velocity vs plan with public dashboards
5. Sponsor confidence is the real metric, not velocity. 1:1 sentiment check with sponsor + key stakeholders weekly; Surface unmet expectations early; Adjust comms cadence to match anxiety level
6. Lock the rescue with a written acceptance. Compile evidence package: diagnostic, plan, results, sentiment; Schedule remediation-acceptance review with sponsor; Obtain signed acceptance + reactivate normal cadence
7. Make sure the next project does not repeat this one. Run a no-blame retro with the full team; Update portfolio risk patterns library; Publish lessons-learned to PMO

## Decision points

- **Diagnose** -> Advance when top 3 root causes are agreed with the delivery lead.
- **Stabilize** -> Advance only when sponsor has signed; do not move on verbal yes.
- **Re-plan** -> Advance when steerco accepts the re-baseline.
- **Execute** -> Stay in Execute until status has been green for 2 weeks.
- **Re-confirm sentiment** -> Advance only if sentiment trend is upward 3 weeks in a row.
- **Sign-off** -> Done when signed acceptance is on file.
- **Lessons** -> Done when doc is published and PMO has acknowledged.
