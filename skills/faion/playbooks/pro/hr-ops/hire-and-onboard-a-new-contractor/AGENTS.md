# Hire and onboard a new contractor (3-5 weeks)

**Playbook slug:** `hire-and-onboard-a-new-contractor`
**Tier:** pro
**Complexity:** medium
**Persona:** P5 -- Micro-agency founder

## Intent

Identified delivery bottleneck to contractor sourced, signed, onboarded, and shipping billable work in week 3.

## Scope

Identify a recurring delivery bottleneck, scope a contractor role, source 5-10 candidates, pick one, sign a contractor agreement, and run them through onboarding so they ship billable work in week 3. Exit artifact: contractor producing acceptable client-facing output with a 30-60-90 plan and clear escalation path.

### What this playbook covers

This is a structured chain of existing faion methodologies adapted for a 2-3 person agency founder who is also the senior delivery operator. It assumes 1-3 contractors handle the rest. Every stage ends with an explicit decision gate so the operator can tell whether to advance, iterate, or kill -- agency founders drift fastest when client comfort overrides honest staging. Each chained methodology lives in the knowledge base and can be read via `faion get-content <methodology-slug>`. The chain order is intentional: skipping a stage typically surfaces as a billing, retention, or contractor problem two months later.

### Non-goals

- W-2 employee hires - this is contractor-only
- Founding-team co-founder search - distinct playbook

### Prerequisites

- At least one recurring engagement type the contractor can absorb
- Bank + invoicing system set up for contractor payments

## Success criteria

The playbook is done when:
- Documented role brief with deliverables + escalation path
- 5-10 candidates screened
- Paid audition completed by 1-3 finalists
- Signed contractor agreement on file (IP + NDA terms locked)
- 30-60-90 day plan delivered to contractor
- Contractor shipping billable client work by week 3

## Stages

### Stage 1: Define the role

**Intent:** Specify the bottleneck and the deliverable shape, not a vague 'helper' role.

**Tasks:**
- Map process bottleneck producing the need
- List 3-5 concrete deliverables for the role
- Define escalation + decision authority

**Methodologies in chain:**
- `business-process-analysis` -> `pro/ba/business-analyst/business-process-analysis`

**Outputs:**

- Bottleneck map

**Decision gate:**
> Advance once a stranger could read the role brief and describe the daily work.

### Stage 2: Source + screen

**Intent:** Get 5-10 plausible candidates into the funnel quickly.

**Tasks:**
- Post role in 3 channels (network, niche board, referrals)
- Screen on portfolio + take-home willingness
- Shortlist 3-5 for paid audition

**Methodologies in chain:**
- `employee-value-proposition` -> `pro/comms/hr-recruiter/employee-value-proposition`

**Outputs:**
- Pipeline of 5-10 candidates
- Shortlist of 3-5

**Decision gate:**
> Advance when shortlist has 3+ candidates above the bar.

### Stage 3: Audition

**Intent:** Paid trial that mirrors actual work; written feedback per candidate.

**Tasks:**
- Issue paid take-home task (4-8h)
- Score against a published rubric
- Run 30-min synthesis interview with top-2

**Methodologies in chain:**
- `interview-methods` -> `pro/comms/hr-recruiter/interview-methods`
- `star-interview-framework` -> `pro/comms/hr-recruiter/star-interview-framework`

**Outputs:**
- Audition rubric scores
- Hire decision doc

**Decision gate:**
> Hire if rubric score above threshold AND finalist comms feel low-friction.

### Stage 4: Contract + comply

**Intent:** Paper everything before the first task; misclassification kills agencies.

**Tasks:**
- Sign contractor agreement with IP + NDA + termination clauses
- Register tax IDs / W-9 / VAT equivalents
- Set up payment + invoicing channel

**Methodologies in chain:**

**Decision gate:**

### Stage 5: Onboard + ship

**Intent:** 30-60-90 plan + week-3 billable output.

**Tasks:**
- Deliver 30-60-90 plan + access kit on day 1
- Pair on first deliverable in week 1
- Hand contractor an independent client task by week 3

**Decision gate:**

## Common pitfalls

- Hiring on 'gut' without a paid audition - fixes the wrong problem
- Skipping the contractor-vs-employee classification check - regulators do not forgive ignorance
- Onboarding by Slack DM - no documented 30-60-90 = no clear ramp

## Quality checklist (self-review)

- Did the paid audition simulate actual client work?
- Is the paperwork done before any client-facing task?
- Does the contractor have a written 30-60-90 plan they can self-check?
