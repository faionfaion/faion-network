# $0 → $4K MRR bootstrap journey

## Intent

Solo dev with shipped MVP grinds to ramen-profitable ($4K MRR / quit-day-job threshold) without VC.

## Scope

Solo dev with shipped MVP grinds to ramen-profitable ($4K MRR / quit-day-job threshold) without VC. Combines distribution flywheels, retention defense, and runway accounting until either MRR target hit or honest sunset.

## Stages

### 1. Baseline + runway

Lock in the financial floor and time budget.

Tasks:
- Compute personal runway months
- Compute current MRR + burn
- Set the quit-day-job MRR threshold with a written trigger contract

Outputs:
- Runway calc sheet
- Baseline MRR
- Quit-day-job trigger contract

Decision gate: Advance only with >=6 months runway. Less than 6 - find part-time income first.

### 2. Single channel bet

Pick ONE acquisition channel and bet hard on it.

Tasks:
- Audit which channel converted past customers
- Score 3 candidate channels by founder-fit + customer-density
- Commit to one channel for the next 90 days

Outputs:
- Channel audit
- Single channel commitment doc
- Weekly KPI per channel

Decision gate: Advance with one named channel and a weekly target. No let us also try X.

### 3. Activation lift

Move the activation rate before pouring traffic on a leaky funnel.

Tasks:
- Map the current activation steps
- Identify the single biggest drop-off
- Ship one onboarding/email intervention

Outputs:
- Activation funnel diagram
- Intervention spec
- Pre/post activation %

Decision gate: Advance when activation rate lifts >=20% relative.

### 4. Retention defense

Plug churn before scaling acquisition.

Tasks:
- Measure 30/60/90 retention cohorts
- Identify top churn reasons via exit survey
- Ship one churn-save email or feature

Outputs:
- Cohort retention chart
- Churn reason ranking
- Save flow spec

Decision gate: Advance when monthly churn drops below the channel growth rate.

### 5. Pricing experiment

Test if you are leaving money on the table.

Tasks:
- Pull pricing competitor research
- Design a price-tier experiment for new signups
- Run for one billing cycle

Outputs:
- Pricing experiment readout
- New tier table OR rollback note

Decision gate: Advance when conversion + ARPU combine to lift MRR per signup >=15%.

### 6. Compounding content/growth loop

Stack a compounding asset on top of the single channel.

Tasks:
- Pick one compounding asset (SEO content, newsletter, referral, community)
- Ship the loop in a measurable form
- Wire metrics so impact is attributable

Outputs:
- Loop documentation
- Weekly compounding metric
- First 4 weeks of data

Decision gate: Advance if the loop produces measurable lift after 4 weeks.

### 7. MRR pulse + dashboard

Weekly visibility so you stop guessing.

Tasks:
- Build a single-screen MRR/churn/activation dashboard
- Schedule a weekly 30-minute pulse review
- Wire alerts on anomalies

Outputs:
- MRR dashboard URL
- Weekly pulse doc template
- Alert rules

Decision gate: Advance when 4 consecutive weekly pulses are done without skipping.

### 8. Trigger evaluation

Honestly check the quit-day-job trigger.

Tasks:
- Compare current MRR to trigger contract
- Stress-test sustainability (3 months of forward MRR)
- Write the quit / keep-job / sunset call

Outputs:
- Trigger evaluation doc
- Sustainability check
- Next-phase commitment

Decision gate: No partial wins. Either trigger met for 3 consecutive months, or restart the loop.

### 9. Sunset OR scale

Either expand the engine or close cleanly.

Tasks:
- If scale: pick the next constraint to attack
- If sunset: refund, notify, archive
- Either way: capture portable lessons

Outputs:
- Scale plan OR sunset comms pack
- Portfolio-strategy update
- Lessons doc

Decision gate: Closes only on a written committed action.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
