# User interview prep and run

## Context

One interview executed with a structured guide, recording and transcript stored, 5-10 atomic insights tagged into a JTBD or opportunity tree within 24 hours.

Tier: **solo**. Complexity: **medium**. Group: **role-product-manager**. Persona: **role-product-manager**.

## Outcome

This playbook is done when:

- Single-line research goal + structured guide.
- Session recorded with consent.
- 5-10 atomic insights tagged within 24h.
- Explicit next-cohort decision.

## Steps

### 1. Prep

Walk into the interview with a single focused goal.

Tasks:
- Write one research question the session must answer.
- Draft a 5-question guide with follow-up probes.
- Confirm recording and consent paperwork.

Outputs:
- 1-question goal + interview guide.

Decision gate: Advance when goal fits on a single line and the guide has follow-up probes.

### 2. Recruit + confirm

Make sure the participant matches the segment and shows up.

Tasks:
- Verify the participant matches the target segment.
- Send the consent + recording note 24h ahead.
- Send a reminder 1h ahead.

Outputs:
- Confirmed participant + consent record.

Decision gate: Advance when the participant has confirmed and consented in writing.

### 3. Run

Run the session as discovery, not sales.

Tasks:
- Open with the participant's recent past, not your product.
- Probe with 'tell me about the last time…'.
- Resist demoing or pitching.

Outputs:
- Session recording + transcript.

Decision gate: Advance when the session ends with the participant having spoken more than the PM.

### 4. Tag insights

Convert raw transcript into atomic, taggable insights.

Tasks:
- Extract 5-10 atomic insights (one fact per line).
- Tag each insight against JTBD and the opportunity tree.
- File any product-affecting insight into the backlog.

Outputs:
- 5-10 tagged insights.

Decision gate: Advance when every insight is one fact (no compound claims).

### 5. Decide next cohort

Use the insight pattern to pick who you talk to next.

Tasks:
- Compare insights against prior 4 interviews.
- Pick one segment to recruit next.
- Adjust the guide if a saturation pattern is emerging.

Outputs:
- Next-cohort decision + guide update.

Decision gate: Required: an explicit next-cohort decision (continue, switch, saturate).

## Decision points

- Continue vs switch cohort — switch once 3 consecutive interviews stop producing new insights.
- Demo vs no-demo — no-demo when the goal is problem space, demo only when the goal is feedback on a build.
