# Monday roadmap pulse review

## Context

Roadmap re-aligned with last week's signals; 2-3 swimlane bets reconfirmed or flagged; deltas posted to Linear/Productboard; async note shipped to founders + EM by end of day.

Tier: **solo**. Complexity: **light**. Group: **role-product-manager**. Persona: **role-product-manager**.

## Outcome

This playbook is done when:

- Half-page weekly signals digest produced.
- Every active bet has a confirm/refute/wait status.
- Async broadcast sent by end of day Monday.
- Drift result recorded for the quarter.

## Steps

### 1. Signal scan

Pull last week's signals before touching the roadmap.

Tasks:
- Skim metrics dashboard for deltas vs target.
- Skim support and sales notes for emerging themes.
- Note any external news (competitor, regulation).

Outputs:
- Signals digest (max half a page).

Decision gate: Advance when the digest fits on half a page.

### 2. Bet review

Re-confirm each swimlane bet against the signals.

Tasks:
- For each active bet, ask: do this week's signals confirm or refute it?.
- Flag bets with refute-signals; reconfirm bets with confirm-signals.
- Tag bets that need more data.

Outputs:
- Bet status table (confirm / refute / wait).

Decision gate: Advance when every active bet has a one-word status.

### 3. Delta application

Apply roadmap deltas in the tracker.

Tasks:
- Move refuted bets to parking lot with reason.
- Promote any confirm-signals item from later to next.
- Re-rank Next by RICE if any new info changed inputs.

Outputs:
- Updated roadmap snapshot.

Decision gate: Advance when tracker matches the bet status table.

### 4. Async broadcast

Send the weekly PM note.

Tasks:
- Draft a 5-bullet broadcast: deltas, asks, blockers, decisions needed, next-week focus.
- Address it to founders + EM + design lead.
- Send by end of day Monday.

Outputs:
- Weekly PM async broadcast (5 bullets).

Decision gate: Advance when broadcast is sent and acknowledged.

### 5. Drift sanity check

Run a quick drift detector against OKRs.

Tasks:
- Compare top-3 in-flight items against the quarter's OKRs.
- Flag any item not tied to an OKR for kill or re-justify.
- Log the drift result for the quarterly retro.

Outputs:
- Drift flag list.

Decision gate: Required: a written drift result (even if zero items flagged).

## Decision points

- Refute vs wait — refute requires acting against a bet; wait means insufficient signal yet.
- Sync vs async broadcast — solo PM defaults to async; sync only when a decision is needed in real time.
