# Customer demo / sneak-peek session

## Context

A 30-minute demo for 3-10 design partners runs cleanly; structured feedback is captured against 3 hypotheses; follow-ups are filed within 48 hours.

Tier: **solo**. Complexity: **medium**. Group: **role-product-manager**. Persona: **role-product-manager**.

## Outcome

This playbook is done when:

- 3 sharp hypotheses with confirm/refute criteria.
- 3+ design partners attended.
- Hypothesis × partner feedback grid completed.
- Follow-ups filed and partners thanked within 48h.

## Steps

### 1. Pick hypotheses

Walk in with 3 explicit hypotheses to test.

Tasks:
- Write 3 sharp hypotheses the demo will pressure-test.
- Map each hypothesis to a moment in the demo flow.
- Define what 'confirm' and 'refute' look like for each.

Outputs:
- 3-hypothesis demo plan.

Decision gate: Advance when each hypothesis has explicit confirm/refute signals.

### 2. Recruit partners

Pick 3-10 partners that span the target use cases.

Tasks:
- Invite design partners matching the target segments.
- Confirm attendance + role + use case.
- Send the recording-consent note.

Outputs:
- Confirmed attendee list.

Decision gate: Advance when 3+ partners are confirmed.

### 3. Run demo

Run the demo as a probe, not a pitch.

Tasks:
- Demo the flow tied to the hypotheses, not the full product.
- Pause after each hypothesis-moment for reactions.
- Avoid 'leading' questions.

Outputs:
- Session recording.

Decision gate: Advance when each hypothesis-moment has a clear partner reaction logged.

### 4. Capture feedback

Tag feedback per hypothesis live.

Tasks:
- Use a feedback grid (hypothesis × partner).
- Mark each cell as confirm / refute / neutral.
- Capture verbatim quotes.

Outputs:
- Hypothesis × partner feedback grid.

Decision gate: Advance when every hypothesis has a row of partner reactions.

### 5. File follow-ups

Convert reactions into product actions within 48 hours.

Tasks:
- Decide per hypothesis: confirmed, refuted, iterate.
- File backlog items per decision.
- Send partners a 'thanks + here's what we heard' note.

Outputs:
- Follow-up actions + partner thank-you.

Decision gate: Required: every hypothesis exits with a confirmed/refuted/iterate verdict.

## Decision points

- Demo vs interview — demo when you have something to react to; interview when you are still problem-finding.
- 1:1 partner sessions vs group demos — 1:1 yields deeper signal; group is faster but partners self-censor.
