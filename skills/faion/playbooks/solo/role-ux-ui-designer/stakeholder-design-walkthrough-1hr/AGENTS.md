# Stakeholder design walkthrough (1hr)

## Context

Designer runs a 1-hour walkthrough with non-designer stakeholders. Covers context, journey, prototype demo, decision extraction, and scope creep parking. Done when decisions are captured and follow-ups assigned.

## Outcome

Misaligned stakeholders -> aligned + decisions extracted + scope creep parked. Non-designer stakeholders aligned on what's being built and why; decisions extracted, scope creep contained, follow-ups assigned.

## Steps

1. **Open with context.** Stakeholders need the why before the what. Tasks: Restate user problem + success metrics; Show the journey at a high level; Set expectations for the walkthrough.
2. **Walkthrough.** Walk the prototype, not the pixels. Tasks: Demo prototype on flagship flows; Surface the trade-offs made; Invite questions at decision points.
3. **Extract decisions.** Force decisions on open questions. Tasks: List open questions live; For each: decide, defer, or park; Capture each with owner + deadline.
4. **Park scope creep.** Big new ideas go to the park, not the sprint. Tasks: Park new asks with rationale; Tie parks to roadmap or next research cycle; Confirm parks with the suggester.
5. **Close + follow-up.** Land the meeting with action. Tasks: Recap decisions + parks + follow-ups; Assign follow-ups with owners + due dates; Send written summary same day.

## Decision points

- **After Open with context:** Advance only when stakeholders can restate the user problem.
- **After Walkthrough:** Advance only when stakeholders followed the walkthrough.
- **After Extract decisions:** Advance only when every open question has a status.
- **After Park scope creep:** Advance when parks are agreed.
- **After Close + follow-up:** Done when summary is sent.
