# Prototype iteration on one flow (1-3hr)

## Context

Designer takes one user flow that has fresh usability feedback and iterates it in 1-3 hours. Includes change list, wireframe update, hi-fi update, interaction wiring, and a rationale log. Done when the prototype is ready for the next test and the rationale entry is filed.

## Outcome

Prototype with usability findings -> updated prototype + logged rationale + ready to retest. One user flow re-prototyped after fresh feedback; updated interactive prototype ready for the next test session; rationale logged.

## Steps

1. **List changes.** Translate findings into concrete edits. Tasks: List each critical finding + proposed change; Flag changes that affect the journey map; Park lower-severity changes for next iteration.
2. **Update wireframes.** Sketch structural changes first. Tasks: Rewireframe affected steps; Resolve information-architecture impacts; Mark steps where copy or affordance changes.
3. **Update hi-fi.** Apply tokens + content updates. Tasks: Apply hi-fi to updated screens; Keep all values on tokens; Update microcopy + affordances.
4. **Wire interactions.** Prototype must run end-to-end. Tasks: Wire links + transitions for the flow; Smoke-test the prototype; Fix dead-ends.
5. **Log + tag.** Future you will thank present you. Tasks: File iteration log with rationale per change; Apply version tag (e.g., v3.2-onboarding); Schedule the next test session.

## Decision points

- **After List changes:** Advance only when changes are concrete (no 'improve UX').
- **After Update wireframes:** Advance only when wireframes match the change list.
- **After Update hi-fi:** Advance only when hi-fi matches wireframes.
- **After Wire interactions:** Advance only when smoke test passes.
- **After Log + tag:** Done when log is filed and tag is applied.
