# User interview prep + run (1 session)

## Context

Researcher runs one participant interview. Includes pre-flight (env, consent, recording), running the interview, hot-take note within an hour, raw artifacts filed in the repo, and a follow-up either scheduled or politely closed. Done when notes are filed and the follow-up state is logged.

## Outcome

Open interview slot -> ran on time + notes filed + follow-up scheduled. One participant interview end-to-end: pre-flight checks, on-time run, notes captured, raw artifacts filed, follow-up scheduled.

## Steps

1. **Pre-flight.** Don't lose data to a flat battery. Tasks: Confirm consent + compensation; Test recording + backup recording; Open script + research repo file.
2. **Open.** First 3 minutes set the tone. Tasks: Introduce yourself + the purpose; Restate confidentiality + recording; Warm-up question.
3. **Probe JTBD + pain.** Get past surface answers. Tasks: Walk current workflow step-by-step; Probe pain points and workarounds; Capture critical incidents.
4. **Wrap.** Close cleanly with consent intact. Tasks: Ask the participant if anything was missed; Confirm compensation + thanks; Set expectation for follow-up.
5. **Hot take.** Fresh notes are best notes. Tasks: Write a 5-min hot take while still fresh; Capture 3-5 key quotes; Note any surprising signals.
6. **File + follow-up.** Make data findable next week. Tasks: Upload recording + transcript to repo; Apply consistent file naming; Schedule follow-up or close politely.

## Decision points

- **After Pre-flight:** Advance only after recording + consent are confirmed.
- **After Open:** Advance only when the participant looks comfortable.
- **After Probe JTBD + pain:** Advance only when critical incidents are described.
- **After Wrap:** Advance when participant has consented to follow-up state.
- **After Hot take:** Advance only after note is written.
- **After File + follow-up:** Done when repo entry exists with all artifacts.
