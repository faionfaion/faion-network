# Research insight synthesis (2hr/week)

## Context

Designer/researcher runs a weekly 2-hour synthesis. Includes review of artifacts, AI-assisted coding, insight write-up with evidence, linkage to design backlog, and a top-theme selection. Done when insights are filed and the next prototype theme is picked.

## Outcome

Raw weekly research artifacts -> 3-7 actionable insights + linked backlog items + next-prototype theme. Last week's research artifacts condensed into 3-7 actionable insights with evidence; insights linked to design backlog items; top theme picked for next prototype.

## Steps

1. **Inventory.** List the week's artifacts. Tasks: Pull interviews, surveys, diary entries; Confirm artifacts are filed + named correctly; Note any incomplete artifacts.
2. **AI coding.** Tag patterns fast. Tasks: Run AI transcript + open-response analysis; Human-validate the top codes; Reject false positives.
3. **Cluster.** Group codes into themes. Tasks: Affinity-cluster codes into themes; Tie themes to JTBD + pain points; Cut weak themes (<3 evidence points).
4. **Insight write-up.** Concrete, evidence-backed insights. Tasks: Write 3-7 insight cards (claim, evidence, implication); Add representative quotes per insight; Cross-check against opportunity tree.
5. **Backlog + theme.** Tie insights to action. Tasks: Link each insight to a design backlog item; Pick top theme for next prototype; Schedule the prototype iteration.

## Decision points

- **After Inventory:** Advance only when inventory is complete.
- **After AI coding:** Advance only after human validation.
- **After Cluster:** Advance only when every theme has >=3 evidence points.
- **After Insight write-up:** Advance only when each card has claim + evidence + implication.
- **After Backlog + theme:** Done when the next prototype theme is picked and scheduled.
