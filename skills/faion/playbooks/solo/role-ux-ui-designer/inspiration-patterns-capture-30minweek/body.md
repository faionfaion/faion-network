# Inspiration + patterns capture (30min/week)

## Context

Designer runs a weekly 30-minute pattern capture. Includes browsing, evaluating, tagging, filing, and queuing a usable pattern for this sprint. Done when at least one pattern is filed with rationale and queued for application.

## Outcome

Stale pattern bank -> 30-min weekly intake + filed + one usable pattern queued. Weekly intake of design patterns, competitor moves, new component libraries; filed by problem-space; one usable pattern queued for application this sprint.

## Steps

1. **Browse.** Cast the net but stay focused. Tasks: Skim 3-5 inspiration sources; Note competitor moves; Capture screenshots of candidates.
2. **Evaluate.** Decide what is signal vs noise. Tasks: Filter candidates against problem-space tags; Note anti-patterns explicitly; Pick at most 3 to file.
3. **File.** Build a queryable bank. Tasks: Tag each pattern by problem-space; Add a 1-line rationale; Link source + screenshot.
4. **Queue one.** Inspiration becomes work. Tasks: Pick the pattern most useful this sprint; Open a design ticket to apply it; Set a check-in for the result.

## Decision points

- **After Browse:** Advance only with candidates captured.
- **After Evaluate:** Advance only when shortlist is <=3.
- **After File:** Advance only after tags are applied.
- **After Queue one:** Done when ticket exists.
