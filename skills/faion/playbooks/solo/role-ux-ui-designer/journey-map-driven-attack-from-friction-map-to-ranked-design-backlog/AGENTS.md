# Journey-map-driven attack: from friction map to ranked design-backlog

## Context

Designer takes a journey map and converts friction into a prioritized backlog. Includes friction audit, debt vs bet split, RICE scoring, ticket creation, and rationale traceability. Done when ranked tickets are in the backlog and traceable to journey-map evidence.

## Outcome

Journey map of friction -> ranked design-debt + design-bet tickets in backlog. Existing or new journey map in hand; ends with a ranked list of design-debt + design-bet tickets pushed into the product backlog with traceable rationale.

## Steps

1. **Read the map.** Find the friction. Tasks: Walk each journey stage; Note pain points + delight gaps; Capture evidence per friction.
2. **Service blueprint.** Show the system behind the journey. Tasks: Map front-stage + back-stage actions; Identify dependencies + failure points; Tag system-level frictions.
3. **Debt vs bet split.** Sort known fixes vs experimental bets. Tasks: Mark each item as debt or bet; Document the difference live with team; Park ambiguous items for refinement.
4. **RICE-for-design.** Rank with a consistent rubric. Tasks: Score reach + impact + confidence + effort; Apply design-specific RICE adjustments; Order by score.
5. **Tickets + trace.** Push to backlog with rationale. Tasks: Open backlog tickets per item; Link each ticket to journey-map evidence; Communicate the new ranked queue.

## Decision points

- **After Read the map:** Advance only when each friction has evidence.
- **After Service blueprint:** Advance only with blueprint reviewed.
- **After Debt vs bet split:** Advance only when every item has a label.
- **After RICE-for-design:** Advance only with ranked list reviewed.
- **After Tickets + trace:** Done when tickets are linked and the queue is shared.
