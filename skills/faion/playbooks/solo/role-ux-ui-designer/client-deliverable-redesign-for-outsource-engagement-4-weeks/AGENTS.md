# Client deliverable redesign for outsource engagement (4 weeks)

## Context

Outsource designer takes an existing client surface across 4 weeks. Includes targeted research, IA + heuristic cleanup, hi-fi redesign on priority flows, validation, and a handoff with KPI scoreboard. Done when client signs off and KPI scoreboard is wired for post-launch measurement.

## Outcome

Inherited client surface -> redesigned priority flows + validated + handoff with KPIs. Outsource agency picks up an existing app or marketing surface, runs targeted research, redesigns priority flows, validates, ships a vendor-friendly handoff with measurable KPIs.

## Steps

1. **Discovery intake.** Understand the client + their constraints. Tasks: Run a discovery questionnaire with the client; Capture brand, tech, and compliance constraints; Audit current surface for content + IA issues.
2. **Targeted research.** Validate the JTBD before redesigning. Tasks: Run 5 JTBD interviews (real users or client-side proxies); Capture top pain points + behaviors; Define success metrics for the redesign.
3. **Heuristic + cognitive walkthrough.** Find the friction worth fixing. Tasks: Heuristic evaluation on priority flows; Cognitive walkthrough for novice scenarios; Pick priority flows for redesign investment.
4. **Redesign.** Wireframes -> hi-fi using tokens. Tasks: Wireframe priority flows; Apply design tokens (use client's system or scaffold one); Build interactive prototype.
5. **Validate.** Test with users before client sign-off. Tasks: Run 5 usability sessions on priority flows; Capture severity-ranked findings; Iterate on critical issues.
6. **Client sign-off.** Land the design + the scoreboard. Tasks: Run readout with client decision-makers; Walk through the design KPI scoreboard; Capture sign-off with success criteria.
7. **Handoff.** Vendor-friendly handoff to client engineering. Tasks: Package spec + tokens + prototype; Hand off to client engineering with a walkthrough; Define post-launch measurement check-in.

## Decision points

- **After Discovery intake:** Advance only when constraints are written and acknowledged.
- **After Targeted research:** Advance when client accepts the JTBD framing.
- **After Heuristic + cognitive walkthrough:** Advance only when client agrees on priority flows.
- **After Redesign:** Advance when prototype walks priority flows without dead ends.
- **After Validate:** Advance only when critical issues are resolved.
- **After Client sign-off:** Advance only when sign-off is written, not verbal.
- **After Handoff:** Done when client engineering accepts the package.
