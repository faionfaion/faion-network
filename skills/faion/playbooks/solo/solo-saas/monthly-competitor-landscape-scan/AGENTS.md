# Monthly competitor & landscape scan

## Intent

Sharper positioning: top 5 competitors re-audited, pricing & feature deltas captured, 1 positioning tweak shipped, kill-list updated for non-differentiating roadmap items.

## Scope

Sharper positioning: top 5 competitors re-audited, pricing & feature deltas captured, 1 positioning tweak shipped, kill-list updated for non-differentiating roadmap items.

## Stages

### 1. Re-list the field

Refresh the top-5 competitor list.

Tasks:
- Pull the current top-5 from search + social + customer mentions
- Drop anyone irrelevant; add new entrants
- Lock the canonical list for this month

Outputs:
- Top-5 competitor list
- Drop/add log
- Canonical list URL

Decision gate: Advance only when exactly 5 competitors are named.

### 2. Feature + pricing audit

Capture deltas in a single sheet.

Tasks:
- Walk each competitors pricing page
- Capture feature deltas vs your product
- Note any new positioning angle

Outputs:
- Feature delta sheet
- Pricing delta sheet
- Positioning notes

Decision gate: Advance only when every competitor row is filled with a date.

### 3. Positioning tweak

Ship one small positioning change before next month.

Tasks:
- Pick one delta worth responding to
- Update headline / pricing card / one section
- Run the change past 3 outsider readers

Outputs:
- Updated landing section
- Outsider feedback notes
- Ship-date stamp

Decision gate: Advance only when one positioning change actually deploys.

### 4. Kill non-differentiators

Cut roadmap items that everyone already has.

Tasks:
- Review backlog for table-stakes items not differentiating
- Mark them kill / accept / defer
- Update the kill log

Outputs:
- Backlog kill list
- Updated kill log
- Capacity freed-up estimate

Decision gate: Advance only when at least one backlog item is killed.

### 5. Positioning diff log

Keep a record of how positioning evolves.

Tasks:
- Write a one-paragraph diff vs last month
- Tag the strongest competitor and why
- Set reminder for next monthly scan

Outputs:
- Positioning diff entry
- Strongest-competitor note
- Next-scan reminder

Decision gate: Cycle closes when the diff log entry is dated and committed.

### 6. Brand-storytelling refresh

Keep the one-liner sharper than competitors.

Tasks:
- Re-write your one-liner once per quarter
- Test it against the top competitors claim
- Roll the winner into landing + bios

Outputs:
- New one-liner
- A/B reasoning
- Rollout note

Decision gate: Advance only when one-liner survives the outsider test.

### 7. Topical-authority signal

Spot SEO gaps the competitors leave.

Tasks:
- Pull the gap keywords competitors rank for that you do not
- Pick one cluster to attack next month
- Queue 1 content brief

Outputs:
- Keyword gap list
- Chosen cluster
- Content brief

Decision gate: Cycle closes when one content brief is filed.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
