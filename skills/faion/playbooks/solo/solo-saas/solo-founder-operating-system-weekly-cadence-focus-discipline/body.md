# Solo founder operating system (weekly cadence + focus discipline)

## Intent

An honest operating rhythm for a one-person SaaS: which day for shipping, which for marketing, which for inbox, plus the kill-switch for context-switching that destroys solo productivity.

## Scope

An honest operating rhythm for a one-person SaaS: which day for shipping, which for marketing, which for inbox, plus the kill-switch for context-switching that destroys solo productivity.

## Stages

### 1. Pick weekly themes

Assign each weekday a single theme.

Tasks:
- Cluster work into themes (ship / market / inbox / admin)
- Assign one theme per weekday
- Reserve at least one deep-work block per ship day

Outputs:
- Weekly theme grid
- Daily theme labels
- Deep-work calendar blocks

Decision gate: Advance only when every weekday has a single dominant theme.

### 2. Revenue-relevance filter

Apply the money filter to every new task.

Tasks:
- Define the revenue-relevance filter (yes / maybe / no)
- Apply it to your current backlog
- Drop or defer no items

Outputs:
- Filter definition
- Filtered backlog
- Drop log

Decision gate: Advance only when filter has cut >=20% of backlog.

### 3. Context-switch protocol

Stop bleeding focus between modes.

Tasks:
- Define the rules for switching modes (max once/day)
- Set up rituals for entering/exiting each mode
- Note the cost of unplanned switches

Outputs:
- Switch rules
- Entry/exit rituals
- Switch-cost log

Decision gate: Advance only when rules are committed and visible.

### 4. Tripwires + burnout signals

Catch burnout before it costs a month.

Tasks:
- Define personal burnout tripwires (sleep, mood, output)
- Set weekly self-check
- Pre-write the response if any tripwire fires

Outputs:
- Tripwire list
- Self-check schedule
- Response plan

Decision gate: Advance only when self-check fires onto the calendar.

### 5. Weekly review ritual

Close the week intentionally.

Tasks:
- Run the solo weekly review (shipped / cut / next)
- Update runway calc + MRR pulse
- Pick next weeks top-3 outcomes

Outputs:
- Weekly review doc
- Updated runway/MRR
- Top-3 next week

Decision gate: Cycle closes when review is committed and next-week is primed.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
