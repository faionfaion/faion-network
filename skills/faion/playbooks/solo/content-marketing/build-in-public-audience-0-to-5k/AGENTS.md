# Build-in-public audience growth: 0 to 5K followers over 6 months

## Why this playbook exists
Indie hackers run the entire pipeline alone — research, build, ship, market, support, and decide — under tight calendar constraints and often from a nomad schedule. Zero audience → 5K followers on X plus 1K newsletter subscribers in 26 weeks. This playbook turns that ambition into a sequenced set of stages with explicit decision gates so the founder doesn't confuse motion for progress and doesn't keep paying for products that quietly stopped working.

## What changes after running it
Solo indie hacker compounds an owned distribution channel — Twitter/X plus a newsletter — from zero to a launch-ready 5K/1K base over six months by running a weekly build-in-public cadence with measurable feedback loops.

## Stage flow
### Positioning
One-sentence persona-niche-promise that every post will reinforce.

**Tasks**
- Write a positioning sentence: 'I help <persona> do <job> via <method>'
- Map the 3 core themes the account will own
- Set up bio, pinned tweet, and newsletter homepage to match

**Decision gate**
> Advance when a stranger can describe what the account is about after one scroll.

### Cadence
A weekly content engine that survives travel and low energy.

**Tasks**
- Define daily tweet slots (build / lesson / signal / banter)
- Schedule one weekly newsletter draft + send cadence
- Set up a hook bank fed by replies and DMs

**Decision gate**
> Advance if cadence survives 2 consecutive weeks unaided.

### Newsletter Funnel
Twitter readers convert to newsletter subscribers at a measurable rate.

**Tasks**
- Build a 1-page newsletter landing with a real promise
- Pin a tweet that pitches the newsletter
- Send issue #1, then every Sunday after

**Decision gate**
> Advance if signup rate ≥3% of weekly profile visits.

### Community
From broadcast to conversation: replies and DMs do the compounding.

**Tasks**
- Set a daily 30-min reply window on 10 in-niche threads
- Run a monthly 'ask me anything' to surface objections
- Activate active-listening on every newsletter reply

**Decision gate**
> Advance if at least 5 inbound DMs/week are starting from your replies.

### Iteration
Monthly OKR review tightens the cadence and themes.

**Tasks**
- Pull follower/subscriber deltas every month
- Cut the 3 lowest-engagement post types
- Reset OKRs for the next 30 days

**Decision gate**
> Advance if monthly deltas trend positive 3 months in a row.
> Re-position if growth flatlines for 2 consecutive months.

## When to skip a stage
Every stage exists because indie hackers historically skip exactly that one — Validate is skipped by builders who fall in love with code, Decide is skipped by founders who can't bear to kill a product, and Cadence is skipped when travel tempts a creator into silence. If you must skip a stage, write a one-paragraph rationale in the playbook artefact log so the next run can audit the trade-off.

## How this playbook compounds
The artefacts from each run — quotes, postmortems, conversion tables, hook banks — feed directly into the next playbook the founder runs. A weekly review fed by these outputs is the difference between a portfolio of half-finished projects and a portfolio of compounding bets. Treat every gap[] entry as a methodology to ship; the playbook stays in draft until the gaps clear.

## Reading order
First-time runs should read the playbook YAML end-to-end before starting Stage 1. Repeat runs can jump to the stage matching current ambiguity and skim only that stage's methodologies. The decision gate is always the source of truth — even if the founder feels confident, the gate decides.

## Failure modes to watch
- Mistaking audience engagement for paying intent; the playbook insists on a paid checkpoint.
- Letting one stage drag because the next is intimidating — the gate is meant to force the move.
- Burning out by trying to combine this playbook with two others in the same week.

## Done means done
The playbook is done when every success criterion is checked AND a written verdict exists in the founder's notes. Anything short of a written verdict is an in-progress playbook — file it back into the queue and keep going.
