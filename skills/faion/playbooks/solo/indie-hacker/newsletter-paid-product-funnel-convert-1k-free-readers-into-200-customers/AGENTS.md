# Newsletter → paid product funnel: convert 1K free readers into 200 customers

## Intent

Engineer a documented funnel that turns a 1K-subscriber free newsletter into ~200 paying customers at $20 average through low-friction email-driven conversion, hitting $4K MRR.

## Scope

Engineer a documented funnel that turns a 1K-subscriber free newsletter into ~200 paying customers at $20 average through low-friction email-driven conversion, hitting $4K MRR.

## Stages

### 1. Map the funnel

See subscribe -> open -> click -> buy as one chart.

Tasks:
- Pull subscribe + open + click metrics
- Plot the funnel
- Mark the biggest leak

Outputs:
- Funnel chart
- Leak label
- Baseline conversion %

Decision gate: Advance only with a single named leak.

### 2. Sharpen positioning

Re-write the paid offer for the subscriber base.

Tasks:
- Re-write the offer headline for the niche
- Match price to the audiences WTP
- Test 3 angles on subscribers

Outputs:
- New offer copy
- Pricing decision
- Angle test results

Decision gate: Advance only when one angle outperforms by >=25%.

### 3. In-issue ad format library

Build repeatable ad placements inside the newsletter.

Tasks:
- Design 3 in-issue ad formats (banner, native, p.s.)
- Slot them into the next 4 issues
- Track CTR per format

Outputs:
- Format library
- 4-week rotation
- CTR per format

Decision gate: Advance only when at least one format hits CTR target.

### 4. Onboarding email sequence

Convert new subscribers in the first 14 days.

Tasks:
- Write a 5-email welcome -> sales sequence
- Trigger on subscribe
- Track open + click + buy per email

Outputs:
- Sequence live
- Per-email metrics
- Buy rate by day

Decision gate: Advance only when sequence converts >=5% of new subs in 14 days.

### 5. Live launch issue

Run a focused conversion push to the whole list.

Tasks:
- Build a launch issue with a single CTA
- Schedule the send + follow-up
- Track revenue per issue

Outputs:
- Launch issue live
- Follow-up issue
- Revenue per send

Decision gate: Advance only when launch issue produces measurable revenue spike.

### 6. Churn-save email pack

Stop new customers from bouncing in month 1.

Tasks:
- Wire churn-trigger emails (no usage, payment fail, downgrade)
- Test 2 variants per trigger
- Track save rate

Outputs:
- Trigger emails live
- Variant test results
- Save rate

Decision gate: Advance only when save rate >=20%.

### 7. Pricing experiments

Test packaging on the engaged segment.

Tasks:
- Design a 3-tier or annual-bundle test
- Roll out to a sub-cohort
- Measure ARPU + conversion

Outputs:
- Pricing experiment spec
- ARPU delta
- Conversion delta

Decision gate: Advance only when test produces net positive revenue lift.

### 8. Weekly cadence + dashboards

Lock weekly reads so the funnel keeps moving.

Tasks:
- Build a single-screen funnel dashboard
- Run a weekly funnel review
- Adjust experiments based on data

Outputs:
- Dashboard URL
- Weekly review log
- Adjusted experiment plan

Decision gate: Advance only when dashboard runs for 4 consecutive weeks.

### 9. Hit the 200-customer mark

Close the loop with the OKR.

Tasks:
- Tally paid customers vs target
- Write the OKR retrospective
- Plan the next funnel evolution

Outputs:
- Customer count
- OKR retro
- Next-funnel plan

Decision gate: Cycle closes when OKR is met OR retro names the next move.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
