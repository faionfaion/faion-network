# Tweet-to-launch sprint: idea to paying user in 3 weeks

## Intent

From a one-line tweeted hypothesis to a Gumroad/Stripe-collected first paying customer, with all build/marketing/ship steps audience-driven and lean.

## Scope

From a one-line tweeted hypothesis to a Gumroad/Stripe-collected first paying customer, with all build/marketing/ship steps audience-driven and lean.

## Stages

### 1. Tweet the hypothesis

Float the one-line problem statement publicly.

Tasks:
- Write a single-tweet problem statement
- Post and tag the audience explicitly
- Track replies + DMs for signal

Outputs:
- Hypothesis tweet
- Audience tag list
- Reply signal log

Decision gate: Advance only with >=10 engagement signals from the target audience.

### 2. Quick Mom-test calls

5 calls before any build.

Tasks:
- DM 5 strong-signal repliers
- Run Mom-test problem interviews
- Capture severity + willingness-to-pay

Outputs:
- 5 call notes
- Severity tally
- Pricing reactions

Decision gate: Advance only with >=3/5 painful + payment signals.

### 3. Ugly first version

Build the smallest demoable thing.

Tasks:
- Cut the scope to the smallest demo-able workflow
- Build with the simplest stack you know
- Deploy to a real URL within days

Outputs:
- Demo URL
- Scope cut log
- First commit history

Decision gate: Advance only when one user can complete the workflow.

### 4. Pre-sell + Gumroad/Stripe

Take money before polish.

Tasks:
- Wire Gumroad or Stripe checkout
- Publish a landing with one button
- Send the link to all warm leads

Outputs:
- Checkout live
- Landing URL
- Outreach log

Decision gate: Advance only when checkout is testable and live.

### 5. Build-in-public ship day

Use the audience to amplify the launch.

Tasks:
- Ship a thread with the journey + link
- Reply to every engagement quickly
- Cross-post a short version to IH

Outputs:
- Launch thread
- Reply log
- IH cross-post

Decision gate: Advance only when ship day is actually shipped.

### 6. First paying user

Close the loop with one real receipt.

Tasks:
- DM warm leads with a personal close
- Take the payment and onboard live
- Capture verbatim feedback

Outputs:
- First paid invoice
- Onboarding notes
- Verbatim feedback

Decision gate: Advance only when at least one paid invoice lands.

### 7. Kill criteria check

Apply the indie kill criteria before sinking more time.

Tasks:
- Score the launch against the kill criteria
- Write the continue / pivot / kill call
- Schedule the next sprint OR sunset

Outputs:
- Kill criteria score
- Decision doc
- Next-sprint or sunset plan

Decision gate: Cycle closes only with a written committed call.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
