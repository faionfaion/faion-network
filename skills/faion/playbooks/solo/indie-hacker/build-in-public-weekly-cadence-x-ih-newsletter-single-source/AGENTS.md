# Build-in-Public Weekly Cadence (X + IH + newsletter, single source)

## Intent

Run a sustainable single-source content cadence — one weekly action that fans out to X, IH, newsletter — without burning out on a nomad schedule.

## Scope

Run a sustainable single-source content cadence — one weekly action that fans out to X, IH, newsletter — without burning out on a nomad schedule.

## Stages

### 1. Pick the weekly anchor

One main piece per week, fan out from there.

Tasks:
- Choose an anchor format (essay, build log, teardown)
- Block a weekly writing slot
- Set the topic for the next 4 weeks

Outputs:
- Anchor format
- Writing slot
- 4-week topic plan

Decision gate: Advance only when slot + topics are on the calendar.

### 2. Single-source draft

Write once, slice many.

Tasks:
- Draft the anchor piece
- Slice into: tweet thread, IH post, newsletter, X repost
- Schedule each slice

Outputs:
- Anchor draft
- Slice variants
- Schedule grid

Decision gate: Advance only when all slices are scheduled.

### 3. Low-energy fallback mode

Stay alive on nomad / bad weeks.

Tasks:
- Pre-write 2 evergreen fallback pieces
- Set a rule: ship fallback if low energy hits
- Track when fallback fires

Outputs:
- Fallback pieces
- Trigger rule
- Fallback fire log

Decision gate: Advance only when fallbacks exist before they are needed.

### 4. Engagement loop

30 minutes of replies before each post.

Tasks:
- List 5-10 accounts to reply to before publishing
- Time-box the engagement to 30 minutes
- Track which replies convert

Outputs:
- Engage list
- Reply log
- Convert delta

Decision gate: Advance only when engagement ritual runs each week.

### 5. Weekly retro + adjust

Compound by reviewing what worked.

Tasks:
- Tally reach, replies, subscribers
- Note which slice format won
- Adjust the next weeks format mix

Outputs:
- Weekly retro doc
- Winning format note
- Next-week adjustments

Decision gate: Cycle closes when retro is committed.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
