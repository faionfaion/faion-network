# Build-in-public audience growth: 0 to 5K followers over 6 months

## Intent

Compound an owned audience (Twitter/X + newsletter) from zero to a 5K-follower / 1K-subscriber base that can be activated for every future product launch.

## Scope

Compound an owned audience (Twitter/X + newsletter) from zero to a 5K-follower / 1K-subscriber base that can be activated for every future product launch.

## Stages

### 1. Niche + positioning

Pick a niche your audience can repeat back.

Tasks:
- Choose a niche tighter than indie hacker
- Write the positioning one-liner
- Set the OKR for follower + subscriber growth

Outputs:
- Niche statement
- Positioning one-liner
- Audience OKR

Decision gate: Advance only when 3 friends can repeat the niche.

### 2. Content engine setup

Build a single-source content pipeline.

Tasks:
- Pick one weekly anchor format (essay, build log, teardown)
- Set the X + IH + newsletter fan-out template
- Pre-build a 4-week buffer

Outputs:
- Anchor format
- Fan-out template
- 4-week content buffer

Decision gate: Advance only when buffer has 4 ready-to-publish items.

### 3. Daily reply discipline

Compound via reply-guy protocol.

Tasks:
- Pick 5-10 accounts to engage daily
- Reply with value before posting your own content
- Track which replies convert to follows

Outputs:
- Engage list
- Daily reply log
- Convert rate per reply

Decision gate: Advance only when reply ritual runs 5x/week for 4 consecutive weeks.

### 4. Newsletter capture

Convert followers to subscribers.

Tasks:
- Wire a high-converting landing for the newsletter
- Lead-magnet for the niche
- A/B test the call-to-subscribe

Outputs:
- Newsletter landing
- Lead magnet
- CTA experiment results

Decision gate: Advance only when newsletter conversion >=2% of profile visits.

### 5. Audience listening

Use the audience as continuous discovery.

Tasks:
- Run weekly polls + DMs for product signal
- Capture verbatim quotes
- Feed insights to product backlog

Outputs:
- Poll bank
- Verbatim quote bank
- Backlog deltas

Decision gate: Advance only when 3 verbatim insights ship to backlog per month.

### 6. Community building

Move best followers from public to private.

Tasks:
- Open a private space (Discord/Telegram/Circle)
- Curate the first 50 members
- Run a monthly event or AMA

Outputs:
- Private community URL
- First 50 members
- Monthly event log

Decision gate: Advance only when monthly event has >=20 attendees.

### 7. Track + iterate against OKR

Weekly review of the audience funnel.

Tasks:
- Pull follower + subscriber growth weekly
- Compare to OKR pace
- Adjust content cadence + topics

Outputs:
- Weekly OKR report
- Cadence adjustments
- Topic backlog

Decision gate: Advance only when weekly review runs without skipping for 8 weeks.

### 8. Activate audience for launch

Prove the audience converts.

Tasks:
- Plan a small product launch to the list
- Measure conversion + revenue
- Capture lessons for the next launch

Outputs:
- Launch plan
- Conversion + revenue numbers
- Launch lessons

Decision gate: Advance only when launch hits its pre-set revenue target.

### 9. Audience-as-moat doc

Lock the asset and protect it.

Tasks:
- Write the audience-as-moat doc
- Inventory the list assets + access
- Plan portability (own list vs platform-only)

Outputs:
- Moat doc
- Asset inventory
- Portability plan

Decision gate: Cycle closes when own-list portability is confirmed.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
