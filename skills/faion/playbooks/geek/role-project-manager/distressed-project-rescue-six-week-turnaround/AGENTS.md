# Distressed-project rescue (six-week turnaround)

## Context

Take over or stabilise a project that has missed two consecutive milestones, lost client trust, or has red on cost/schedule/quality. Exit with a credible recovery plan executing for 3+ weeks.

Tier: **geek**. Complexity: **deep**. Group: **role-project-manager**. Persona: **role-project-manager**.

## Outcome

This playbook is done when:

- Top-3 root causes named with evidence.
- Client confirms accurate understanding of state.
- Rebaselined plan signed.
- 3+ consecutive weeks of positive trend.
- Documented hand-back to steady-state PM.

## Steps

### 1. Diagnose week 1

Find the root cause(s) within 5 working days.

Tasks:
- Interview client, team, and outgoing PM.
- Audit schedule, cost (EVM), and quality data.
- Write a 1-page diagnosis with top 3 root causes.

Outputs:
- Diagnosis 1-pager.

Decision gate: Advance when 3 root causes are named with evidence, not speculation.

### 2. Stabilise comms

Stop the trust bleed.

Tasks:
- Send a candid, factual update to the client.
- Schedule a daily check-in for the first 2 weeks.
- Acknowledge known issues without overpromising.

Outputs:
- First candid update.
- Daily client check-in cadence.

Decision gate: Advance when the client confirms they understand the situation.

### 3. Rebaseline scope

Cut scope honestly before promising recovery.

Tasks:
- Identify cuttable scope without breaking client value.
- Rebaseline scope + schedule + cost.
- Get written sign-off on the new baseline.

Outputs:
- New baselined scope + plan.

Decision gate: Advance when the new baseline is signed by the client.

### 4. Re-team

Adjust the team for the rescue, not the original plan.

Tasks:
- Identify capability gaps.
- Swap or add team members where critical.
- Re-run team-development rituals to reset chemistry.

Outputs:
- Re-teamed roster + reset rituals.

Decision gate: Advance when the new team has worked one full cycle together.

### 5. Execute weeks 2-5

Run the rescue with daily ops and weekly client demos.

Tasks:
- Daily standup + weekly demo to client.
- Track via kanban or value-stream board.
- Burn down rebaselined backlog visibly.

Outputs:
- Weekly demos + burn-down chart.

Decision gate: Advance when 3+ consecutive weeks show positive trend.

### 6. Earned trust check

Reconfirm trust before declaring recovery.

Tasks:
- Run a candid trust check with the client.
- Document trust signals (renewed scope, references, follow-on work).
- Decide rescue continue, hand back, or sunset.

Outputs:
- Trust check note + decision.

Decision gate: Required: a written trust verdict at week 6.

### 7. Lessons + handover

Capture the rescue lessons before exiting.

Tasks:
- Run a structured lessons-learned with the team.
- Write a recovery playbook for the org.
- Hand the project back to a steady-state PM.

Outputs:
- Lessons-learned doc.
- Steady-state handover note.

Decision gate: Required: a documented hand-back with steady-state PM named.

## Decision points

- Cut scope vs add resources — cut scope first; throwing people at a late project usually makes it later.
- Rescue vs hand back vs sunset — hand back when trust is unrecoverable but value remains; sunset when neither holds.
