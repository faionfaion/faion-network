# Sunset + rebuild from a failed launch

## Intent

Turn a missed launch into a clean shutdown plus a sharper next-bet plan: refund/migrate users, harvest learnings publicly, recycle assets, decide on rebuild vs pivot vs walk away.

## Scope

Turn a missed launch into a clean shutdown plus a sharper next-bet plan: refund/migrate users, harvest learnings publicly, recycle assets, decide on rebuild vs pivot vs walk away.

## Stages

### 1. Honest debrief

Write a brutally honest post-mortem.

Tasks:
- Fill the indie post-mortem template
- Capture what worked, what failed, what surprised
- Tag the dominant cause

Outputs:
- Post-mortem doc
- Tagged causes
- Surprise list

Decision gate: Advance only when the cause is named, not vibed.

### 2. Customer + community comms

Protect trust with everyone who showed up.

Tasks:
- Draft a public what-happened post
- Send personal emails to paying customers
- Process refunds / migrations

Outputs:
- Public post
- Customer emails sent
- Refund/migration log

Decision gate: Advance only when every paying customer has a personal touch.

### 3. Asset harvest

Salvage the parts worth keeping.

Tasks:
- Inventory code, copy, audience, learnings
- Tag each: keep / archive / kill
- Move keepers to a next-bet folder

Outputs:
- Asset inventory
- Keep folder
- Archive/kill log

Decision gate: Advance only when every asset has a tag.

### 4. Public learnings post

Turn the failure into audience compounding.

Tasks:
- Write a long-form learnings post
- Cross-post to X / IH / newsletter
- Engage with every comment

Outputs:
- Learnings post
- Cross-post log
- Engagement log

Decision gate: Advance only when post is live AND replied to.

### 5. Decide rebuild vs pivot vs walk

Apply kill criteria, not nostalgia.

Tasks:
- Score against the indie kill criteria
- List candidate next bets
- Pick: rebuild / pivot / walk

Outputs:
- Score sheet
- Candidate list
- Decision call

Decision gate: Advance only when one path is named in writing.

### 6. Plan the next move

Write a tight plan for whichever path won.

Tasks:
- If rebuild: spec the new v1
- If pivot: open the new-bet sprint
- If walk: wind down hosting, close accounts

Outputs:
- Next-move plan
- Wind-down actions (if applicable)
- First-week tasks

Decision gate: Cycle closes when first week is scheduled.

## Common pitfalls

- Skipping the decision-gate write-up to keep moving - closes the loop with vibes, not evidence.
- Treating each stages outputs as optional - every output is a gate input for the next stage.
- Letting one bad week stretch a fixed-cadence ritual into a quarterly one.

## Quality checklist

- Every stage has at least one referenced methodology that resolves under `knowledge/`.
- Every output is a real artefact, not a feeling.
- The final decision is a written commitment, not we will see.
