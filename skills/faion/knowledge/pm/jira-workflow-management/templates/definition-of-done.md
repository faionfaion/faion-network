# Definition of Done — [Project Name]

Apply to every Story/Bug before transitioning to Done.

## Code

- [ ] Code complete and pushed to branch
- [ ] Branch linked to Jira issue (smart commit or manual link)
- [ ] Code review approved by at least one reviewer
- [ ] No critical or blocker-severity comments unresolved

## Testing

- [ ] Unit tests written (coverage threshold: [N]%)
- [ ] Integration tests pass in CI
- [ ] QA sign-off on acceptance criteria
- [ ] No open P1/P2 bugs against this issue

## Documentation

- [ ] Inline code comments for non-obvious logic
- [ ] API changes documented (OpenAPI/changelog)
- [ ] User-facing changes reflected in release notes

## Acceptance

- [ ] Product Owner accepted the deliverable
- [ ] Deployed to staging environment and verified
