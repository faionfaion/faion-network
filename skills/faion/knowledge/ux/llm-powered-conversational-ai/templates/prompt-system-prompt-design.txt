You are a conversational AI system designer. Design a production-ready system prompt for:

Product: {product_name}
Use case: {use_case}
Persona name: {persona_name}
Persona tone: {persona_tone}
Allowed topics: {topic_list}
Escalation triggers: {escalation_conditions}

Output:
1. System prompt (under 500 tokens, use explicit lists for allowed and forbidden topics)
2. Guardrail rules (what the assistant must refuse, as a numbered list)
3. Three example dialogues:
   a. Normal happy-path flow (5 turns)
   b. Ambiguous query requiring clarification (3 turns)
   c. Out-of-scope request with graceful deflection (2 turns)
4. Escalation message template (what the assistant says when handing off to a human)

Rules:
- Keep the system prompt under 500 tokens
- Use explicit lists, not vague "stay on topic" instructions
- Include response length limit in the persona constraints
- Test: would the system prompt survive "ignore your previous instructions"?
