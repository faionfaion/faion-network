# Visibility of System Status

## Summary

**One-sentence:** Nielsen Heuristic #1 applied: keep users informed about what is going on through appropriate feedback within a reasonable amount of time — every state change has a visible indicator, every long action shows progress, every async outcome is announced.

**One-paragraph:** Nielsen Heuristic #1 applied: keep users informed about what is going on through appropriate feedback within a reasonable amount of time — every state change has a visible indicator, every long action shows progress, every async outcome is announced. The methodology pins the artefact: a rubric scoring the UI on instant feedback, progress indication, async outcome announcements, and accessibility-friendly status messaging.

**Ефективно для:**

- UIs with async actions (upload, save, generate).
- Reviewers catching silent failures and frozen states.
- Accessibility reviewers checking aria-live regions and status roles.
- Audit surface: rubric score per screen.

## Applies If (ALL must hold)

- UI has any async action (≥200ms perceived latency).
- Users wait for outcomes generated by the system.
- Accessibility matters (screen readers in audience).

## Skip If (ANY kills it)

- Pure-read UI with no state changes.
- Latency is consistently <100ms; instant feedback is implicit.
- Audience is purely API-only.
