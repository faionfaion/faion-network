# Enterprise XR Applications

## Summary

Methodology for designing XR (AR/VR/MR) applications for enterprise contexts: long-session comfort, tool integration, security, accessibility compliance, and ROI-measurable outcomes. Enterprise XR differs from consumer XR in that it must fit existing workflows, run for hours without discomfort, and meet corporate IT requirements.

## Why

Enterprise XR deployment fails most often on non-UX factors: IT incompatibility, insufficient session comfort for multi-hour training, missing data security controls, and inability to prove ROI. Designing with these constraints as first-class requirements prevents costly late-stage rework.

## When To Use

- Designing XR training simulations, remote assistance overlays, or digital twin interfaces for industrial/enterprise use
- Evaluating whether an XR solution is appropriate for a specific enterprise use case
- Defining requirements for enterprise XR procurement or vendor selection
- Reviewing an existing consumer XR experience for enterprise adaptation needs

## When NOT To Use

- Consumer-facing XR games or entertainment apps — enterprise constraints (IT security, session length, ROI metrics) do not apply
- 2D web or mobile enterprise software — separate methodology applies
- Early-stage research on whether XR is needed at all — use a simpler feasibility framework first
