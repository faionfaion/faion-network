# Spatial Interaction Patterns

## Summary

Reference for the five input modalities in spatial/XR interfaces (hand tracking, controllers, gaze, voice, gesture) and the three primary interaction patterns (direct manipulation, ray-casting, gaze+dwell) with their respective use cases, strengths, and accessibility implications.

## Why

Spatial interfaces support multiple simultaneous input modalities; choosing the wrong primary modality for a task leads to arm fatigue (gorilla arm), precision errors, or excluded user groups. Mapping task type to input modality early in design prevents these problems at prototype stage rather than after development.

## When To Use

- Designing interaction flows for an XR application
- Selecting primary and fallback input modalities for a spatial feature
- Evaluating an existing XR interaction for accessibility (gaze+dwell as motor-impairment fallback)
- Writing interaction specifications for XR developers

## When NOT To Use

- 2D touch or mouse-based interfaces — different interaction model
- Voice-only VUI design — use error-handling-in-vui or vui-accessibility-inclusivity instead
- Platform selection decisions — use spatial-computing-overview instead
