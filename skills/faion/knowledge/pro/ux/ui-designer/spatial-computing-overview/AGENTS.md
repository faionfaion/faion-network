# Spatial Computing Overview

## Summary

Reference overview of spatial computing platforms, terminology, and market context for designers entering AR/VR/MR work. Covers the distinction between AR, VR, and MR; the current platform landscape (Apple Vision Pro, Meta Quest, Android XR, HoloLens, Magic Leap); and the design implications of 3D-space interfaces versus rectangular screens.

## Why

Designers coming from 2D backgrounds often conflate AR, VR, and MR, leading to wrong platform assumptions, incorrect interaction model selection, and deliverables that do not match the target device's capabilities. A shared vocabulary and platform map prevents misaligned design decisions early in a project.

## When To Use

- Onboarding to a spatial computing project with no prior XR design experience
- Choosing which XR platform to target for a given use case
- Writing briefs or technical specs that need precise AR/VR/MR terminology
- Evaluating market context and platform maturity before committing to a platform

## When NOT To Use

- Deep-dive on specific interaction patterns — use the spatial-interaction-patterns methodology instead
- Tooling decisions — use the spatial-design-tools methodology instead
- Enterprise XR requirements — use the enterprise-xr-applications methodology instead
