# Spatial Design Tools

## Summary

Tool selection guide and design-to-development workflow for spatial (XR) design. Covers the gap between 2D design tools (Figma) and native spatial tools (ShapesXR, Gravity Sketch), and defines a four-stage workflow from concept through production: 2D wireframes → low-fi headset prototype → high-fi Unity/Unreal → production.

## Why

Spatial design cannot be adequately specified or tested in traditional 2D tools. Key interaction qualities (depth, scale, arm reach, comfort) are invisible in Figma mockups and only manifest in a headset. A staged workflow that introduces the headset at the low-fi prototype stage catches spatial UX problems before costly high-fidelity development.

## When To Use

- Selecting tools for a new spatial/XR design project
- Setting up a design-to-development handoff process for an XR team
- Deciding when to move from 2D to 3D in the design process
- Recommending tool investments to stakeholders

## When NOT To Use

- 2D screen-based UI design — standard Figma/Sketch workflow applies
- XR content production (3D modeling, animation) — separate tools and skills
- Enterprise XR requirements specification — use enterprise-xr-applications instead
