# 60-90 Day Onboarding Phases

## Summary

**One-sentence:** A two-phase framework for structured new-hire progression after the initial 30-day learning phase: Contribute (days 31-60, first owned project, active collaboration) and Execute (days 61-90, independent delivery, measurable results).

**One-paragraph:** A two-phase framework for structured new-hire progression after the initial 30-day learning phase: Contribute (days 31-60, first owned project, active collaboration) and Execute (days 61-90, independent delivery, measurable results). Each phase ends with a milestone checkpoint anchored to a concrete artifact, not a self-reported activity.

## Applies If (ALL must hold)

- A hire has cleared 30-day learning criteria and needs explicit Contribute (31-60) and Execute (61-90) milestones.
- Manager wants a structured second-and-third-month plan separate from the broad 30-60-90 framework.
- Roles with ramp quotas or feature ownership (sales, engineering) where deliverables land in this window.
- Preparing a defensible 90-day review with an artifact trail.

## Skip If (ANY kills it)

- Hire is still failing day-30 criteria — resolve or restart the cycle first.
- Roles with multi-quarter ramps (Enterprise AE, ML researcher) — extend phases; do not compress.
- Senior autonomous hires who define their own deliverables.
- Contract placements under 60 days.
