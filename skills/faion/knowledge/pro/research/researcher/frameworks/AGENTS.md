# Research Frameworks: A Router for Pre-Revenue Decisions

## Summary

**One-sentence:** A toolbox of eight named pre-revenue research frameworks — 7Ps ideation, Paul Graham questions, niche evaluation (50-point matrix), validation criteria, Jobs-to-be-Done, TAM/SAM/SOM, competitive intelligence, and Value Proposition Canvas — treated as a router: the agent picks the smallest subset that answers the current decision, not all eight at once.

**One-paragraph:** A toolbox of eight named pre-revenue research frameworks — 7Ps ideation, Paul Graham questions, niche evaluation (50-point matrix), validation criteria, Jobs-to-be-Done, TAM/SAM/SOM, competitive intelligence, and Value Proposition Canvas — treated as a router: the agent picks the smallest subset that answers the current decision, not all eight at once.

## Applies If (ALL must hold)

- Founder at the "what should I build?" stage needing a structured idea funnel.
- Investor pitch requiring TAM/SAM/SOM plus a competitive table.
- Pivot review: re-score current niche against the same matrix to make stay/pivot defensible.
- GTM repositioning: rebuild the Value Proposition Canvas before rewriting the homepage.
- Multi-idea triage: score N ideas in parallel and rank by niche score.

## Skip If (ANY kills it)

- Already shipping with paying customers — use cohort analysis and retention curves instead.
- Single-question clarifications ("should I charge $19 or $29?") — use a pricing experiment.
- Pure technical scoping (architecture, API design) — use SDD design.md.
- Hobby projects, internal tools, OSS side projects with no revenue goal.
- Two-sided marketplaces pre-launch where supply/demand dynamics dominate.
