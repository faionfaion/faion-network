# Software Architect Domain Skill

## Purpose

Pro-tier architectural decision support for distributed, cloud-native systems. Covers cloud architecture, container orchestration, data pipelines, event-driven and microservices patterns, plus cross-cutting concerns: observability, reliability, and quality attributes.

## When to Use

| Scenario | Methodology |
|----------|-------------|
| Designing for AWS/GCP/Azure | cloud-architecture |
| Kubernetes / scheduler topology | container-orchestration |
| Streaming / batch ETL design | data-pipeline-design |
| Picking distributed-system patterns | distributed-patterns |
| Async, decoupled services | event-driven-architecture |
| Decomposing into services | microservices-architecture |
| Logs, metrics, traces strategy | observability-architecture |
| Trade-off analysis (perf vs cost vs sec) | quality-attributes-analysis |
| SLA, fault tolerance, DR | reliability-architecture |
| Sidecar, mTLS, traffic policy | service-mesh |
