## Event Storming Discovery Prompt

Run Event Storming on <spec>. Output a markdown event catalog with columns:
  Event | Producer | Consumers | Trigger | Schema fields | Retention (days) | Ordering required | Idempotency key

Rules:
- Reject CRUD-shaped names (UserUpdated, OrderModified). Past tense + business outcome only.
- Minimum 5 events.
- Every event must have at least 2 fields beyond standard CloudEvents envelope.
- Every consumer must list its idempotency key field.

## Saga Generation Prompt

Given the multi-step process in <spec>, design as orchestration-based saga.

Emit:
1) workflow.py with each step as an activity call
2) compensations.py with one compensation per step
3) failure_modes.md listing what happens if step N fails after step N-1 has succeeded

Rules:
- Refuse to emit a workflow without all compensations present.
- Each compensation must be idempotent.
- Name compensation methods cancel_<step> to make intent clear.
- Add a comment on each step explaining its compensation.
