# North Star Metric

## Summary

**One-sentence:** A framework for identifying, validating, and operationalizing a single metric that captures the core value a product delivers to customers — predicts long-term revenue, aligns all teams, and serves as the root of the OKR tree.

**One-paragraph:** A framework for identifying, validating, and operationalizing a single metric that captures the core value a product delivers to customers — predicts long-term revenue, aligns all teams, and serves as the root of the OKR tree. Includes a 4-step selection process, validation test, real-product examples, and input-metrics tree pattern.

## Applies If (ALL must hold)

- Teams are misaligned on what success looks like day-to-day.
- Marketing optimizes for clicks, product for features, sales for revenue — no shared growth lever.
- Setting up OKRs and need a root metric that team-level key results should roll up to.
- Scaling growth investment and need to validate that acquisition spend improves the right outcome.

## Skip If (ANY kills it)

- Pre-launch with no usage data — the NSM requires retention/revenue correlation to validate.
- Very early (solo, pre-PMF) stage — pick one proxy metric and iterate; formal NSM process adds overhead before it helps.
- When the product serves two fundamentally different user types with different value definitions — a single NSM will be dishonest for one group.
