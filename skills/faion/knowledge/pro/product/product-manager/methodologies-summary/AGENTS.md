# Methodologies Summary (Product Manager)

## Summary

A routing index for all 33 methodologies in the `pro/product/product-manager/` skill. Not a methodology itself — it answers "which methodology do I need?" for a given PM task, mapping tasks to slugs in a Quick Selection Guide. Read this first when entering product-manager scope without a specific methodology pinned; then load the chosen methodology's own `AGENTS.md`.

## Why

With 33 methodologies spanning MVP scoping, feature prioritization, roadmaps, discovery, business models, and operations, an agent entering the skill cold has no cheap way to pick the right methodology without a router. This summary provides a one-read decision tree that narrows the search to a single slug before any heavy content is loaded.

## When To Use

- Cold start: agent has a PM task and no methodology is pre-selected.
- Task triage: deciding between RICE vs. MoSCoW, MVP vs. MLP, roadmap vs. OKR, lifecycle vs. feedback management.
- Onboarding a new PM or agent to the skill — first read to orient.

## When NOT To Use

- Methodology already chosen — skip summary, go straight to that folder's `AGENTS.md`.
- Cross-skill work (BA, PM-agile, project-manager) — use the respective skill's own router.
- Execution or templates — look in the chosen methodology's own files, never here.
