# Team Development

## Summary

**One-sentence:** A framework for building high-performing project teams using Tuckman's stages (Forming → Storming → Norming → Performing → Adjourning) as the diagnostic backbone, paired with three artifact types: team charter, skills matrix, and retrospective template.

**One-paragraph:** A framework for building high-performing project teams using Tuckman's stages (Forming → Storming → Norming → Performing → Adjourning) as the diagnostic backbone, paired with three artifact types: team charter, skills matrix, and retrospective template. The PM adapts leadership style per stage and uses tracked metrics (PR attribution, contribution distribution) to move the team through stages deliberately.

## Applies If (ALL must hold)

- Onboarding a new hire or contractor and wanting to map them onto Tuckman stages.
- Sprint retro reveals interpersonal conflict (Storming) and PM needs structured mediation steps.
- Solo founder hiring first 1–3 people and needing a written charter, skills matrix, and integration plan.
- Tracking a team's progression so leadership knows when to push autonomy vs. provide direction.

## Skip If (ANY kills it)

- One-off async working groups disbanding in a week — Tuckman overhead exceeds value.
- Pure individual-contributor work with no inter-dependency.
- Highly transactional vendor relationships — Tuckman assumes a sustained team.
