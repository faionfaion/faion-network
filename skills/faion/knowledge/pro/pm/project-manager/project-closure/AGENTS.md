# Project Closure

## Summary

**One-sentence:** A formal 7-step process to end a project: verify deliverable acceptance with signed artefacts, close financial and contract obligations, conduct lessons learned before the team disperses, archive all documentation, produce a final report, hand over to operations, and recognise the team.

**One-paragraph:** A formal 7-step process to end a project: verify deliverable acceptance with signed artefacts, close financial and contract obligations, conduct lessons learned before the team disperses, archive all documentation, produce a final report, hand over to operations, and recognise the team. Closure applies equally to normal completion, cancelled projects, and phase-end gates in multi-phase programmes. Every checklist item is a record with status, owner, and evidence — not a checkbox that an agent can self-certify.

## Applies If (ALL must hold)

- Project objectives met and final deliverables accepted (normal closure).
- Project cancelled mid-flight — still close formally to recover value, release resources, capture lessons.
- Phase-end of a multi-phase programme before the next phase receives funding.
- Agency engagement final invoice and warranty handover.
- Absorption of a legacy project into BAU operations.

## Skip If (ANY kills it)

- Project still has open scope, unresolved issues, or unsigned acceptance — resolve those first.
- Operational or run-rate work with no defined endpoint — use service transition documents instead.
- Long-tail support work after handover — that is operations, not project closure.
