Tool naming pre-ship checklist
==============================

[ ] Name is lowercase snake_case
[ ] Name is exactly verb_object (one verb, one object)
[ ] Verb is action-y (create/read/update/delete/search/list/send/cancel)
[ ] Object is concrete (issue, file, payment) — not "thing" / "data"
[ ] No action enum kwarg hiding multiple verbs (split into atomic tools)
[ ] If catalog has tools from > 1 service: namespace prefix added
[ ] Total length <= 40 chars (ideally <= 24)
[ ] Name does not collide with any other tool in the same agent context
[ ] After rename: re-run tool-selection eval, keep the higher score
