# AI-Generated IaC Review Gate

## Summary

**One-sentence:** A pre-merge gate for any IaC diff authored or substantially edited by an AI assistant — runs four mandatory checks (drift, security, performance, multi-arch) and refuses to merge until each clears or a human consciously waives it.

**One-paragraph:** AI-generated Dockerfiles, Helm charts, Terraform modules, and GitHub Actions look right and break in prod. The four common failure shapes: (1) drift — the diff references a resource that doesn't exist or has different shape than expected; (2) security — exposed ports, root containers, missing pod-security-standards, unrestricted IAM; (3) performance — wrong instance class, missing requests/limits, sync calls in async contexts; (4) multi-arch — `FROM amd64-only` or apt packages that don't exist on arm64. This methodology defines a checked-into-repo gate (`iac-gate.yaml`) that runs in CI on every IaC-touched PR. The gate emits a report; the diff cannot merge until every check is `pass` or `waived: <reason>`.

## Applies If (ALL must hold)

- Repository contains IaC files (Dockerfile, *.tf, *.tfvars, Helm chart, kustomize, GHA workflows).
- The diff was authored or last edited by an AI assistant (commit author / co-author / agent-session marker).
- A CI runner is available to execute the four checks.
- The repo is destined for production deployment, not a sandbox.

## Skip If (ANY kills it)

- Diff is documentation-only inside an IaC file (e.g. comment update).
- Sandbox / experiments-only repo with no production target.
- IaC change is generated by a deterministic tool (Renovate, Dependabot) rather than an LLM.
- A stricter sibling gate is already enforced (`ai-iac-hallucination-detector` + `ai-generated-iac-review-redflags` combined) — coordinate to avoid double-gate.

## Prerequisites

- `iac-gate.yaml` checked into the repo with the four checks enabled.
- Provider credentials available to CI for plan-only (no apply) runs.
- A baseline `arch-matrix.yaml` listing supported architectures (amd64, arm64, etc.).
- Pod-security baseline (PSA) declared at the cluster or namespace level.
