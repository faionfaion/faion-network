# Agent Integration — AI Plugin Ecosystem (Figma)

## When to use
- Evaluating which Figma AI plugins to adopt for a specific team workflow
- Automating repetitive design tasks: bulk renaming, content population, icon generation
- Generating a plugin adoption policy with accessibility and brand guardrails
- Auditing design files for missing content, broken links, or accessibility violations at scale

## When NOT to use
- Final production asset export — AI plugin outputs require human QA before handoff
- Design system creation — plugins assist but cannot own system architecture decisions
- Small one-off tasks where plugin setup time exceeds manual effort (fewer than ~20 components)
- Contexts where plugin API access to external services raises data privacy concerns (e.g., healthcare data in designs)

## Where it fails / limitations
- Most Figma plugins have no stable public REST API — agents cannot invoke them headlessly; Figma's plugin sandbox is browser-JS only
- AI plugin outputs (Magician, etc.) vary with each run — reproducibility is poor; not suitable for deterministic pipelines
- Plugins like Content Reel pull live data; network failures silently produce empty states
- Accessibility plugins (Stark) flag contrast issues but cannot audit interaction flow or focus order
- Plugin ecosystem is fragmented: no standard for how plugins consume or emit design tokens
- Figma REST API v2 does not expose plugin state — you cannot query "which plugins are active" programmatically

## Agentic workflow
Agents cannot directly invoke Figma plugins (sandbox restriction), but they can: (1) generate plugin configuration scripts and batch operation recipes for a human to run, (2) analyze Figma file exports (JSON via REST API) to identify opportunities for plugin automation, (3) produce plugin evaluation matrices scored against team criteria. The human runs plugins in Figma; the agent pre-processes and post-processes.

### Recommended subagents
- `haiku` — generating bulk operation recipes, plugin configuration checklists, content data files for Content Reel
- `sonnet` — plugin ecosystem evaluation matrices, accessibility gap analysis from Figma JSON export, adoption policy drafts

### Prompt pattern
```
You are a Figma plugin workflow designer. Given this team context: [team size, design system maturity, accessibility requirements].
Evaluate these plugins: Magician, Automator, Content Reel, Stark.
Output: scored matrix (criteria: setup cost, output reliability, accessibility coverage, brand safety) + recommended adoption order.
```

```
Given this Figma file JSON export (truncated): [paste nodes].
Identify: (1) layers likely missing real content (lorem ipsum, placeholder images), (2) components with no accessibility annotations, (3) frames suitable for batch AI generation.
Output as actionable task list.
```

## Templates & scripts
Generate a Content Reel data file (JSON) from a structured list — agent can produce this, human imports to plugin:

```python
import json

def generate_content_reel_data(names: list[str], emails: list[str]) -> str:
    """Produce Content Reel-compatible JSON for user profile content."""
    entries = [
        {"Name": n, "Email": e, "Avatar": f"https://i.pravatar.cc/150?u={e}"}
        for n, e in zip(names, emails)
    ]
    return json.dumps({"data": entries}, indent=2)

# Usage: paste output into Content Reel > Custom Data
sample = generate_content_reel_data(
    names=["Alice Martin", "Bob Chen", "Carla Rossi"],
    emails=["alice@co.com", "bob@co.com", "carla@co.com"]
)
print(sample)
```

## Best practices
- Set brand guidelines (color variables, typography styles) in Figma before running any generative plugin — AI outputs anchor to existing styles when present
- Run Stark contrast audit on every screen before handoff; do not rely on visual inspection alone
- Version-pin plugin usage: note plugin version in design system changelog when a plugin modifies components at scale
- Validate AI-generated icons (Magician) against brand voice before accepting — they skew toward generic/clip-art
- Use Automator batch operations for mechanical tasks (rename layers, detach instances) but test on a duplicate file first
- Document which content in a file is AI-generated with a Figma annotation — aids future audits

## AI-agent gotchas
- Agents will propose API-driven plugin automation that does not exist — Figma plugin sandbox is browser-JS only, not server-accessible
- LLM output for "write a Figma plugin" is often stale (Plugin API v2 changes); always validate against https://www.figma.com/plugin-docs/
- Accessibility remediation suggestions from agents (color values, ARIA labels) require a human to apply in Figma — no write API for component props
- Agent-generated Content Reel data files may produce culturally biased placeholder names/content; review for i18n/diversity before use
