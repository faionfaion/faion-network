# AI-Generated Layout Review Checklist

## Summary

**One-sentence:** Ten-point checklist for reviewing AI-generated mockups (v0, Galileo, Figma Make) before they enter a design system or ship to engineering.

**One-paragraph:** Ten-point checklist for reviewing AI-generated mockups (v0, Galileo, Figma Make) before they enter a design system or ship to engineering. This methodology codifies the rules, output contract, failure modes, and decision tree needed for a checklist produced by an agent applying ai-generated layout review checklist. The deliverable is validated against an explicit JSON Schema and routed through a decision tree that maps observable signals to rule ids in `01-core-rules.xml`.

**Ефективно для:**

- Building a reproducible checklist for ai-generated layout review checklist across teams.
- Reviewing AI-or-human work against an explicit contract instead of vibes.
- Wiring the output into downstream automation (CI gates, observability, post-mortems).
- Avoiding the failure modes listed in `03-failure-modes.xml`.

## Applies If (ALL must hold)

- mockup generated by an AI tool (v0.dev, Galileo, Figma Make, Magician, Uizard, Visily) targeting production_screen or design-system token
- design system with named tokens (color, type, spacing, radius, motion) is already in place
- reviewer is a designer or designer-lead human (not the engineering hand-off step)

## Skip If (ANY kills it)

- mockup is exploratory only (sketch, moodboard, divergence phase) and will not ship
- design system does not exist yet — use design-foundations-bootstrap first
- engagement is a one-off audit, not ongoing — use enterprise design-review process
