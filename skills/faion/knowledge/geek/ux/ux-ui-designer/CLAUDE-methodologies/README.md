# UX/UI Methodologies

## Overview

32 core UX methodologies covering usability heuristics, research methods, and design techniques. Each file follows a consistent structure: Problem, Solution, Best Practices, Checklist.

## File Summaries

### Nielsen's 10 Usability Heuristics (10 files)

| File | Title | Key Principle |
|------|-------|---------------|
| visibility-of-system-status.md | Visibility of System Status | Always keep users informed about what is happening through timely feedback |
| match-real-world.md | Match Between System and Real World | Use familiar language, follow real-world conventions, present info logically |
| user-control-freedom.md | User Control and Freedom | Provide undo/redo, clear exits, support user autonomy |
| consistency-standards.md | Consistency and Standards | Follow platform conventions, maintain internal consistency |
| error-prevention.md | Error Prevention | Eliminate error-prone conditions, provide confirmations for critical actions |
| recognition-over-recall.md | Recognition Rather Than Recall | Make objects, actions, options visible; minimize memory load |
| flexibility-efficiency.md | Flexibility and Efficiency of Use | Accelerators for experts, personalization options |
| aesthetic-minimalist.md | Aesthetic and Minimalist Design | Remove irrelevant information, prioritize essential content |
| error-recovery.md | Help Users Recognize and Recover from Errors | Plain language errors, precise problem indication, constructive solutions |
| help-documentation.md | Help and Documentation | Searchable help, task-focused, concise steps |

## Usage

Each methodology file includes:
- **Problem**: The challenge the methodology addresses
- **Solution**: How to apply the methodology
- **Best Practices**: Do's and don'ts
- **Checklist**: Verification items

Reference by file name when applying to design work (e.g., "Apply error-prevention methodology").
