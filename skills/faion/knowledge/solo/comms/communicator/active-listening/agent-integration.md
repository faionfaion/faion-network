# Agent Integration — Active Listening

## When to use
- Generating interview scripts that follow RASA: open questions, paraphrase prompts, silence cues
- Coaching prep: producing reflective-listening response starters keyed to a specific emotional context
- Transcript post-processing: identifying where the interviewer talked too much (>20% of words) or used closed questions
- Requirements gathering support: converting a stakeholder conversation into structured observations using RASA labels

## When NOT to use
- Real-time active listening — this is a human relational skill; agents can scaffold it but cannot perform it live
- Replacing empathic presence: the Level 5 "empathic listening" in the framework requires reading emotional state from tone and body language, which text-based agents cannot do from transcripts alone
- High-stakes conflict conversations — using an agent-generated script in a sensitive interpersonal situation risks mechanical delivery that backfires

## Where it fails / limitations
- Active listening is fundamentally reactive; agents cannot adapt to what they hear in real time during a live conversation
- Paraphrasing generated by an agent without hearing the original tone will often miss the emotional register
- The "3-5 second silence" technique has no text equivalent; agents cannot enforce pauses in asynchronous communication
- Open-question generation tends to produce questions that are correctly open but lack the authentic curiosity that makes them feel natural
- Agents analyzing transcripts for "planning response" barriers cannot reliably distinguish this from considered reply based on text alone

## Agentic workflow
A subagent receives a conversation goal (e.g., "requirements gathering session with a skeptical stakeholder") and outputs: (1) a RASA-structured question flow, (2) 5-10 paraphrase/reflection starters tuned to expected emotional register, (3) a closed-to-open question conversion for any pre-existing question list. After the human conducts the conversation, the agent receives the transcript and annotates it: each turn labeled by RASA step, word-count ratio check, and barrier identification.

### Recommended subagents
- General Claude Sonnet call — question generation and transcript annotation; sufficient for RASA tagging
- `faion` (communicator domain) — for cross-methodology context (mom-test, stakeholder-communication)

### Prompt pattern
Question script generation:
```
Generate a RASA-structured question flow for: <CONVERSATION_GOAL>.
Interviewee profile: <ROLE/CONTEXT>.
Output format:
- Receive: [opening, set attention]
- Appreciate: [acknowledgment phrases]
- Summarize: [paraphrase starters]
- Ask: [5 open follow-up questions]
No yes/no questions. No "Is..." or "Do you..." openers.
```

Transcript annotation:
```
Annotate this conversation transcript.
For each interviewer turn: label as R/A/S/A (RASA), note if it's a closed question,
note if the interviewer speaks more than the interviewee in any 3-turn window.
Transcript: <TRANSCRIPT>
```

Word-ratio checker (measures whether interviewer dominated):
```python
def speaker_ratio(transcript: list[dict]) -> dict[str, float]:
    """
    transcript: list of {"speaker": "A"|"B", "text": "..."}
    Returns word share per speaker as float 0-1.
    """
    from collections import defaultdict
    counts: dict[str, int] = defaultdict(int)
    for turn in transcript:
        counts[turn["speaker"]] += len(turn["text"].split())
    total = sum(counts.values()) or 1
    return {k: round(v / total, 3) for k, v in counts.items()}

# Usage: if ratio["INTERVIEWER"] > 0.20 → flag as dominated
```

## Best practices
- Build the question script before the conversation, but mark questions as "adaptable" — the script is a scaffold, not a reading list.
- For requirements gathering, use Mirroring (repeat last 1-3 words as a question) as a low-effort, high-yield technique; it is the easiest for a human to apply naturally without a script.
- Run the "closed-to-open" conversion on any existing requirement spec questions before a stakeholder session — most specs contain implicit yes/no questions.
- After the conversation, use the transcript annotation pass to identify which RASA steps were skipped (usually Summarize); feed that gap back into coaching notes.
- The "ask for advice or just vent?" technique from the Barriers section prevents misaligned responses; make it a default opener in sensitive conversations.

## AI-agent gotchas
- Agents generating "open" questions often produce questions that start with "What" or "How" but are still leading ("What problems did you have with X?" implies X caused problems). Check for embedded assumptions.
- Transcript word-count analysis does not account for sentence density — a human may say 20 words but pack in 3x the information of an agent's 60-word turn.
- Empathic reflection statements generated without knowing the interviewee's emotional context can land as condescending or inaccurate ("It sounds like you're frustrated..." when they were calm).
- Human checkpoint required: before acting on a synthesized requirements list derived from a transcript, a human must validate that the RASA annotations match the actual intent of the conversation.
- Silence (the 3-5 second pause) is the most effective active listening tool and the one completely absent from agent-driven text exchanges; compensate with explicit "Is there anything you'd like to add?" prompts.
