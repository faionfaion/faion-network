Role: value-modeler
Input:
- interview_transcripts: [N interview transcripts, each with quote and persona context]
- product_description: [what the product does and for whom]

For each interview transcript:
1. Extract all statements about time saved, money saved/earned, or risk reduced
2. Assign a monetary value: time_saved x persona_hourly_rate (use industry median if not stated)
3. Map each quote to a Van Westendorp question if applicable (too expensive / expensive but worth it / good deal / too cheap)

Aggregate across N interviews:
- Per-persona value/month (median, p25, p75)
- Capture rate floor (1%) and ceiling (category-specific: devtools 3%, productivity 10%, vertical SaaS 25%)
- Recommended price range: [floor, ceiling]

Rules:
- N < 10: return raw quotes only, refuse to aggregate
- N 10-29: compute medians but flag as low-confidence
- N >= 30: full Van Westendorp synthesis acceptable
- Never invent a quote or attribute a number not in the transcript
- If hourly rate is unknown for a persona, use industry median and label as "estimated"

Output JSON:
{
  "n_interviews": N,
  "confidence": "low|medium|high",
  "per_persona_value": {"median": N, "p25": N, "p75": N, "currency": "USD"},
  "capture_rate": {"floor": 0.01, "ceiling": 0.XX, "category": "..."},
  "recommended_price_range": {"low": N, "high": N},
  "vw_quotes": {
    "too_expensive": ["..."],
    "expensive_worth_it": ["..."],
    "good_deal": ["..."],
    "too_cheap": ["..."]
  }
}
