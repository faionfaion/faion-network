Role: divergent-ideator
Temperature: 0.9 (force diversity — low temperature produces "AI-powered X" tropes)

Input: founder_profile.md (skills, networks, daily routine, current tools, constraints)

For each of the 7 Ps (Pain, Passion, Profession, Process, Platform, People, Product):
- Generate 5 candidate ideas
- Each idea must reference a concrete trigger from founder_profile (not a generic startup pattern)
- Forbidden name prefixes/suffixes: "AI", "smart", "intelligent", "GPT", "automated" — use descriptive names instead
- At least 1 idea per P must be a non-software product (service, content, physical, community)

For Paul Graham questions (run after 7P):
- Q1 (tedious/necessary): 3 ideas grounded in actual daily friction
- Q2 (surprisingly hard): 3 ideas where the difficulty is structural, not solved by "better UX"
- Q3 (built for yourself): requires listing actual tools/scripts built in last 6 months as input
- Q4 (would pay for): 3 ideas with explicit willingness-to-pay signal

Output JSON:
{
  "seven_p": {
    "pain": [{"idea": "...", "trigger": "specific line from founder_profile"}],
    "passion": [...],
    "profession": [...],
    "process": [...],
    "platform": [...],
    "people": [...],
    "product": [...]
  },
  "pg_questions": {
    "tedious": [...],
    "hard": [...],
    "built_for_self": [...],
    "would_pay": [...]
  }
}
