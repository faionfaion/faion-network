# MLP Planning

## Summary

Plan the upgrade from a validated MVP to a Minimum Lovable Product: a four-layer
audit (Functional → Reliable → Usable → Delightful) that identifies the specific gap
between "it works" and "I love it". MLP = MVP + targeted delight in high-frequency,
high-visibility interactions. The process ends with explicit completion criteria tied
to retention and NPS targets, not feature counts.

## Why

MVPs validate ideas but fail to retain users. Competing products in crowded markets
are already "good enough", so users try an MVP and leave for better UX. Retention
requires emotional connection, and the four-layer model gives a concrete, auditable
path to it: delight cannot be bolted on randomly — it must be layered on top of solid
function, reliability, and usability.

## When To Use

- MVP is live, core hypothesis validated, but Day-7/30 retention is weak.
- Users complete the core flow but don't return or recommend.
- You're preparing a second-wave release specifically targeting retention.
- NPS is below 30 and you need a structured diagnosis.

## When NOT To Use

- Problem hypothesis is still unvalidated — validate with MVP first.
- No usage data yet; MLP audit requires actual interaction patterns to rate layers.
- B2B/enterprise deal where retention is driven by contract, not experience.
- Prioritising more features over polishing existing ones is a strategic choice — revisit after feature bets resolve.
