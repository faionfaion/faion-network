# purpose: Template helper for API Testing (prompt-generate-tests.txt).
# consumes: see content/02-output-contract.xml inputs for api-testing
# produces: artefact conforming to content/02-output-contract.xml
# depends-on: content/01-core-rules.xml + content/04-procedure.xml
# token-budget-impact: ~200-1000 tokens when loaded as context
Read openapi.yaml. For each operation in tags=[{tags}], generate a
pytest function in tests/contract/test_{resource}.py that asserts:
  - 200/201 happy path with example payload from spec
  - 401 when Authorization header is missing
  - 403 when role lacks required scope (use fixture `low_priv_token`)
  - 422 with malformed body (drop one required field)
  - 429 after 11 calls in 1 second (rate limit)

Use openapi-core for response-schema validation on every response.
Do not invent endpoints or response shapes not declared in the spec.
Output: unified diff only. No explanation prose.
