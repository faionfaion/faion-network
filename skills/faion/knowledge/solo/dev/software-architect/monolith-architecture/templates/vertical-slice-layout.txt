Vertical Slice Architecture — Python/Django directory layout
Each feature owns its full stack. Shared code lives only in shared/.

project_root/
├── manage.py
├── config/
│   ├── settings.py
│   ├── urls.py
│   └── wsgi.py
│
├── shared/                     # Cross-cutting concerns only
│   ├── auth/                   # JWT validation, permission classes
│   ├── models/                 # Abstract base models (timestamps, soft-delete)
│   ├── middleware/             # Request ID injection, structured logging
│   └── exceptions.py           # Base exception hierarchy
│
├── orders/                     # Bounded context: Orders
│   ├── create_order/
│   │   ├── __init__.py
│   │   ├── handler.py          # Django View / DRF APIView
│   │   ├── command.py          # Use case: CreateOrderCommand + handler
│   │   ├── repository.py       # ORM data access for this feature
│   │   ├── dto.py              # Request/response serializers
│   │   └── tests/
│   │       ├── test_handler.py
│   │       └── test_command.py
│   ├── cancel_order/
│   │   ├── handler.py
│   │   ├── command.py
│   │   └── tests/
│   ├── models.py               # Django models shared within the orders context
│   └── urls.py                 # Routes for all order features
│
├── inventory/
│   ├── reserve_stock/
│   │   ├── handler.py
│   │   ├── command.py
│   │   └── tests/
│   ├── release_stock/
│   │   └── command.py          # Can be called by other features (event handler)
│   ├── models.py
│   └── urls.py
│
├── payments/
│   ├── charge_card/
│   │   ├── handler.py
│   │   ├── command.py
│   │   └── tests/
│   └── models.py
│
└── notifications/
    ├── send_email/
    │   └── command.py          # Called via event, no HTTP handler
    └── models.py

Rules:
- A feature folder contains ONLY the code for that feature.
- Cross-feature dependencies go through models.py (read-only reference by ID)
  or through domain events (async, no direct import).
- shared/ has NO business logic — only infrastructure abstractions.
- Tests live next to the feature code, not in a top-level tests/ directory.
