Generate a Django service function following the HackSoft Django Style Guide.

CONTEXT:
- Django version: {django_version}
- App: {app_name}
- Entity: {entity_name}
- Action: {action_name}  (e.g. create, update, cancel, publish)
- Description: {description}

REQUIRED OUTPUT:
1. Service function in apps/{app_name}/services.py
2. Domain exceptions in apps/{app_name}/exceptions.py (if new ones needed)
3. Unit test in apps/{app_name}/tests/test_services.py
4. Factory updates in apps/{app_name}/tests/factories.py (if new models needed)

MANDATORY RULES — follow exactly, no exceptions:
- Function name: {entity_name}_{action_name} (snake_case)
- All arguments except the first are keyword-only (after *)
- Decorate with @transaction.atomic for any DB write
- Inside atomic block: use task.delay_on_commit() NOT task.delay()
- Raise ApplicationError subclasses, never Http404 or APIException
- No ORM queries in service functions — call selectors for reads
- No business logic in views or serializers

FORBIDDEN:
- Positional arguments beyond the first
- Returning None on error (raise instead)
- Direct model.objects.filter() in service (use selector)
- task.delay() inside @transaction.atomic

EXAMPLE SHAPE:
```python
@transaction.atomic
def {entity_name}_{action_name}(*, user: User, {param}: {type}) -> {entity_name}:
    # 1. Validate / fetch via selectors
    # 2. Business logic
    # 3. DB write
    # 4. Post-commit side effects (delay_on_commit)
    return result
```

OUTPUT FORMAT: unified diff or full file. No explanation prose.
