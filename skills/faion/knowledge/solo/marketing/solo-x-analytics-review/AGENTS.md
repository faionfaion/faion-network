# Solo X Analytics Review

## Summary

**One-sentence:** A 20-minute weekly X / Twitter analytics review for an indie operator growing an audience — fixed metrics, fixed format, one experiment per week.

**One-paragraph:** `plausible-analytics` covers site traffic, not platform analytics. An indie hacker using X as the primary audience channel needs a specialised weekly review that surfaces what kind of post the algorithm is rewarding for THEM right now, separates outliers from baseline, and converts the review into exactly one written experiment for the following week. This methodology fixes the five metrics (impressions, profile visits, follows, replies-from-strangers, link clicks), the outlier-detection rule, and the experiment-naming convention.

## Applies If (ALL must hold)

- X / Twitter is the primary audience channel
- the operator posts at least 4 times / week
- there are at least 4 weeks of data to compare against
- the operator wants audience-growth or product-pipeline outcomes (not vanity follows)

## Skip If (ANY kills it)

- the operator does not post regularly — fix posting cadence first
- the audience is fully developer-deep and X is a side-channel only (probably LinkedIn or newsletter is primary)
- the operator pays for ads on X — different methodology (paid review)
- the account is dormant or in dispute — pause rather than half-measure

## Prerequisites

- access to X Premium / analytics dashboard (built-in metrics)
- a written posting goal (e.g., +200 qualified followers / month)
- a spreadsheet or Notion page where past weekly reviews are stored
- a short list of "qualified follower" signals (bio mentions ICP, etc.)
