# Feedback Framework: Observation → Principle → Impact → Suggestion

Use this structure for every piece of critique feedback. All four elements are required except Suggestion (optional).

## Structure

**Observation:** I notice that [specific, describable element or behavior]
**Principle/Goal:** Our goal is [stated design goal] / Research shows [principle]
**Impact:** This might [positive or negative consequence] because [mechanism]
**Suggestion (optional):** What if we [specific alternative]?

---

## Worked Example

**Observation:**
I notice the error message appears at the top of the form.

**Principle:**
Users expect error messages near the field that caused the error (NNG: proximity principle).

**Impact:**
Users may scroll down to fix the error but cannot see what the message said, increasing error recovery time.

**Suggestion:**
What if we showed inline validation errors adjacent to each problematic field?

---

## What Good Feedback Is NOT

- "I don't like the colors." — preference, no principle, not actionable
- "Make it pop." — vague, no observation, no mechanism
- "Why didn't you do it the standard way?" — judgment, not a question, no goal anchor
- "This whole layout is wrong." — scope too large, no specific observation
