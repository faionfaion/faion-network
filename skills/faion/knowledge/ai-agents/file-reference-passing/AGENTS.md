# File-Reference Passing Between Pipeline Steps

## Summary

**One-sentence:** Passes file paths, doc IDs, and URLs between agent stages instead of content, cutting cross-stage prompt tokens 50-90%, enabling lazy loading, and surviving process restarts via reference stability.

**One-paragraph:** When agent step N analyses a corpus, return only references (file paths, IDs, URLs, doc-keys) — not the content itself. Step N+1 loads from those references on demand. The corollary: the boundary between agent steps is a list of POINTERS, not a payload of CONTENT. Four benefits compound: context economy, lazy loading, recoverability across restarts, and cheap audit/debugging.

**Ефективно для:** будь-якого мульти-стадійного пайплайну, де кожен крок аналізує корпус і передає результати наступному кроку — від map-reduce над документами до subagent звітів.

## Applies If (ALL must hold)

- Multi-stage pipeline where each stage operates on a corpus.
- Map-reduce or supervisor-worker patterns over documents.
- Subagents return a slim summary that the parent might want to dereference.
- More than 2K tokens of context would otherwise be ferried between stages.

## Skip If (ANY kills it)

- Single-shot agents — no downstream step exists.
- Content small enough that lazy loading adds more overhead than it saves.
- Refs unstable (URLs that expire, IDs that get reassigned) without snapshotting.
- Downstream model lacks tools to load by ref.

## Prerequisites

| Input artifact | Format | Source |
|---|---|---|
| Manifest of corpus items | List of `{id, path, one-line-desc}` | Upstream indexer |
| Stable ref format | `path:relative/file.md`, `db:items/1234`, `gh:owner/repo@sha:path` | Convention |
| Ref-load tool surface | `read_file`, `db_get`, `http_get` | Tool config |

## Decision tree

See `content/06-decision-tree.xml`. The root question is whether more than 2K tokens of context would cross a stage boundary. The tree then picks one of four shapes (scan-then-load, subagent+refs, manifest-pick, recursive descent) based on whether the corpus is flat, tree-shaped, or generated by a subagent.
