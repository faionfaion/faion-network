# Go Standard Layout — Agentic Workflow and Prompts

## Summary

**One-sentence:** Produces a four-stage agentic pipeline for Go service scaffolding: layout-agent (skeleton) → code-gen-agent (resources) → wiring-agent (main.go) → review-agent (import rules). Prompts checked into `.aidocs/`.

**Ефективно для:**

- New Go services generated by LLM agents.
- Multi-resource codebases where boilerplate dominates writing time.
- Teams enforcing a layout via prompts + review gates.
- Solo dev needing repeatable scaffold flow.

**One-paragraph:** Drive Go service scaffolding as a four-stage pipeline: a layout agent generates the skeleton, a code-gen agent emits handler / service / repository / model per resource, a wiring agent assembles `main.go` with DI, and a review agent runs the import-rule checklist. Pair with persistent stack decisions in `.aidocs/` so agents do not re-decide per PR.

## Applies If (ALL must hold)

- Agent SDK or Claude Code available.
- Layout choice already committed to `.aidocs/`.
- Linter (depguard) enforces layer rules.
- CI runs `go vet` + tests.

## Skip If (ANY kills it)

- One-off prototype — manual scaffold is fine.
- Existing legacy layout — refactor first, then automate.
- Languages other than Go — pattern doesn't transfer cleanly.
