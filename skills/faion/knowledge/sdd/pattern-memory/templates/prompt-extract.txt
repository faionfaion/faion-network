== PATTERN EXTRACTION PROMPT ==========================================

After completing feature: {feature-NNN-name}

Analyze the implementation and identify patterns worth preserving.

Input:
- Files changed: [paste git diff or file list]
- Code review findings: [paste review notes]
- Constitution.md: [paste relevant sections to check against]

For each pattern candidate, answer:
1. Is it specific enough to be actionable? (not "write clean code")
2. Is it reusable across future features? (not one-off business logic)
3. Is it consistent with constitution.md? (if not, should constitution be updated?)
4. What confidence level? Low (first time seen) / Medium (2nd time) / High (3rd+ time)

Output each pattern in pattern-entry.md format.
Output each mistake in mistakes-file.md format.

== PATTERN AUDIT PROMPT ===============================================

Audit the patterns.md file: [paste patterns.md]

For each pattern:
1. Last confirmed date — is it more than 6 features ago? Flag for verification.
2. Confidence level — is High still warranted? Has a newer pattern superseded it?
3. Example — is the code path still in the codebase? Check if file still exists.
4. Anti-pattern — is the anti-pattern still commonly tempting? Or already rare?

Output:
- STALE: {pattern-name} — {reason, suggest archive or verify}
- SUPERSEDED: {pattern-name} → {new pattern-name}
- OK: {pattern-name} — still valid
