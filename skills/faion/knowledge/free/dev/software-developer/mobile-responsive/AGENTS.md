# Mobile-Responsive Web Design

## Summary

**One-sentence:** Mobile-responsive web design: viewport configuration, CSS media queries, flexible layouts, touch-friendly interactions, and mobile-first development for devices from 320px to 2560px.

**One-paragraph:** Mobile-responsive web design: viewport configuration, CSS media queries, flexible layouts, touch-friendly interactions, and mobile-first development for devices from 320px to 2560px.

## Applies If (ALL must hold)

- Any public-facing web application or website (desktop, tablet, mobile).
- Progressive web apps (PWA) targeting mobile-first users.
- Content-driven sites (blogs, news, documentation) where mobile traffic is significant.
- APIs with web UI dashboards accessed from phones and tablets.
- E-commerce sites where mobile conversion is critical.

## Skip If (ANY kills it)

- Intranet-only tools for desktop users only (pure desktop apps, VPN-gated software).
- Native mobile apps (iOS/Android) don't need responsive web CSS.
- Hardware-restricted embedded interfaces where screen size is fixed.
- Temporary prototypes or throwaway tools with no long-term maintenance.
