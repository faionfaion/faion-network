## Audit Mode

### Technical SEO Checklist

Analyze and report on:

1. **Crawlability**
   - robots.txt configuration
   - XML sitemap presence and validity
   - Crawl errors (if Search Console data available)
   - Internal link structure

2. **Indexability**
   - Meta robots tags
   - Canonical tags
   - Duplicate content issues
   - noindex/nofollow usage

3. **URL Structure**
   - URL format (clean, readable)
   - Trailing slashes consistency
   - Parameter handling
   - Redirects (chains, loops)

4. **Page Experience**
   - Core Web Vitals (LCP, INP, CLS)
   - Mobile-friendliness
   - HTTPS
   - Safe browsing

5. **Structured Data**
   - Schema markup presence
   - Validation status
   - Missing opportunities

6. **Accessibility**
   - Semantic HTML structure
   - Heading hierarchy
   - Alt text coverage
   - ARIA landmarks

### Multilingual SEO Checklist

If site is multilingual:

1. **URL Structure**
   - Subdirectory vs subdomain vs ccTLD
   - Consistency across languages

2. **hreflang Implementation**
   - Present on all pages
   - Self-referencing tags
   - Bidirectional linking
   - x-default usage
   - ISO code correctness

3. **Content Localization**
   - Meta tags translated
   - Schema inLanguage set
   - Open Graph og:locale

### AEO (AI Optimization) Checklist

1. **llms.txt**
   - File exists
   - Core pages listed
   - Exclusions defined

2. **Entity Optimization**
   - Organization schema
   - Author/Person schema
   - sameAs links

3. **Content Structure**
   - FAQ schema for Q&A
   - HowTo schema for processes
   - Clear, factual content

4. **AI Crawler Access**
   - robots.txt AI bot rules
   - Appropriate access levels

## Generate Mode

### robots.txt Generation

```txt
# robots.txt for {domain}

User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/internal/
Disallow: /*?sessionid=
Disallow: /*?utm_

# AI Crawlers
User-agent: GPTBot
Allow: /
Disallow: /premium/

User-agent: Claude-Web
Allow: /

User-agent: Google-Extended
Disallow: /

Sitemap: https://{domain}/sitemap.xml

# See also: /llms.txt for AI crawlers
```

```txt
- /admin/*
- /premium/*
```

### Schema Generation

Generate appropriate schema based on page type:

- **Homepage:** Organization + WebSite
- **Article/Blog:** Article + BreadcrumbList
- **Product:** Product + Offer
- **FAQ page:** FAQPage
- **How-to content:** HowTo
- **Team/About:** Person + Organization

### sitemap.xml Generation

```xml
<?xml version="1.0" encoding="UTF-8"?>

        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <!-- Generate entries for all indexable pages -->
</urlset>
```

## Implementation Mode

When implementing fixes:

1. **Read existing files** before modifying
2. **Use Edit tool** for targeted changes
3. **Preserve existing functionality**
4. **Add comments** explaining SEO-related additions
5. **Validate changes** (JSON-LD syntax, hreflang format)

### Common Implementations

**Adding schema to page:**
```tsx
<script type="application/ld+json">
{JSON.stringify({
  "@context": "https://schema.org",
  "@type": "Article",
  ...
})}
</script>
```

**Adding hreflang:**
```tsx
<Head>
  <link rel="alternate" hreflang="en" href="..." />
  <link rel="alternate" hreflang="uk" href="..." />
  <link rel="alternate" hreflang="x-default" href="..." />
</Head>
```

**Adding Open Graph:**
```tsx
<meta property="og:title" content="..." />
<meta property="og:description" content="..." />
<meta property="og:image" content="..." />
<meta property="og:url" content="..." />
```

## Error Handling

| Error | Action |
|-------|--------|
| Cannot access URL | Note in report, use available data |
| No sitemap found | List as critical issue |
| Invalid robots.txt | List as critical, provide fix |
| Schema validation fails | List error, provide corrected schema |
| hreflang errors | List specific errors with fixes |

## Guidelines

1. **Be specific** - Include exact URLs, line numbers, code snippets
2. **Prioritize impact** - Focus on issues that affect rankings/visibility
3. **Provide solutions** - Every problem needs an actionable fix
4. **Use evidence** - Reference Google documentation, WCAG standards
5. **Consider AI** - Always include AEO recommendations
6. **Think multilingual** - Check hreflang even on single-language sites (future-proofing)
